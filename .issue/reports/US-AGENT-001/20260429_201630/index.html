叙事: 用户首次使用 BridgeAI，创建自己的 AI Agent，配置基本信息和偏好。

环境阻塞

无法启动验收环境 — 6 种启动方案全部失败

验收需要启动 pnpm dev 后端服务器 + 移动端，但项目依赖存在系统性损坏。

方案命令结果错误
1pnpm --filter server dev失败tsx 符号链接损坏，dist/cli.mjs 不存在
2pnpm add -D tsx失败符号链接仍指向空目录
3npx tsx src/index.ts失败tsx 自身 dist 目录为空
4node dist/index.js失败编译不完整，@bridgeai/shared 类型缺失
5tsc --project apps/server失败100+ 编译错误（DuplicateIdentifier 等）
6jest agentService.test.ts失败70 测试套件全部失败（ts-jest 解析错误）

根本原因

  • pnpm hoisting 失效 — node-linker=hoisted 配置下，大量依赖包未正确提升到根目录
  • 545 个 .js/.ts 混合文件 — 混合存在，且 .js 文件为旧编译结果，引用了不存在的类型
  • 符号链接指向空目录 — apps/server/node_modules/tsx → ../../node_modules/.pnpm/tsx@4.21.0/.../tsx，目录不存在
  • 关键类型缺失 — VisibilityLevel 从 @bridgeai/shared 无法正确解析（profileService.js:6）
