# Story Playbook: 配置用户广告偏好

**Story**: `AD-001` | **Narrative**: 用户配置 Agent，告诉 Agent 自己对什么类型的广告感兴趣。

## 验收旅程概览

本剧本模拟真实用户 "小林" 首次使用 AgentAd 配置广告偏好的完整流程。从进入场景、配置品类偏好、设置预算范围、调整品牌倾向，到配置信息可见性策略、测试 Agent 匹配行为，形成一条有机用户体验路径。

## 前置条件

1. 已安装 BridgeAI mobile app (iOS/Android)
2. 已有 BridgeAI 账号并完成基础认证
3. 数据库已迁移并包含测试用户数据
4. 服务端运行: `pnpm dev` (apps/server, apps/mobile)
5. 测试账号: `test+ad@bridgeai.com` / `password123`
6. 后台已存在若干商家的广告 Agent (用于匹配测试)

### Phase 2: 品类偏好配置

- 验证 AC: `AD-001~func` 用户配置 Agent：品类、预算、品牌偏好

**Step 2.2**: 添加自定义品类

- 验证 AC: `ISSUE-C002~func` L1/L2 层数据结构定义
- 截图: `04-自定义品类.png`

**Step 2.3**: AI 智能提炼品类偏好 (可选)

- 操作:
  1. 在自然语言输入框输入: "我最近刚装修房子，想买一些智能家居产品"
  2. 点击 "AI 提炼" 按钮
- 预期:
  - 显示加载状态 "正在分析..."
  - 提炼结果:

    - 推断偏好: 智能家居、性价比

  - 一键应用到当前配置
- 验证 AC: `US-AGENT-002-AC-4` Agent 从对话中自动提取关键字段并分类
- 验证 AC: `US-AGENT-002~func` 用户与 Agent 对话后成功创建需求和供给

# 1. 启动数据库

pnpm db:migrate

# 3. 种子数据 (如需要)

1. **自然用户流**: 从场景进入 -> 配置 -> 保存 -> 匹配 -> 调整，遵循真实用户操作路径
2. **非 1:1 映射**: 单个步骤验证多个 AC，避免机械对应
