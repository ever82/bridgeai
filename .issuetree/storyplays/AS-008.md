# Story Playbook: Agent 互动与对话

**Story**: `AS-008`
**Narrative**: 用户通过 Agent 进行日常互动、获取信息、完成任务。

## 验收旅程概览

从"主人与 Agent 私聊建立需求"出发，经历"Agent 自主联系候选 Agent"、"主人实时收到私聊建议并干预"、"与对方主人进入四人群聊协商"，完整覆盖 Agent 生命周期中的私聊建议、身份显示、模式切换、披露策略、引荐机制等核心能力。

## 前置条件

- 服务已启动：`pnpm dev`
- 数据库已迁移：`pnpm db:migrate`
- 已完成注册登录，首页可见
- 测试账号同时持有"甲方 Agent"和"乙方 Agent"两个身份（可通过同一用户的两个场景配置，或两个测试账号实现）

## 旅程步骤

---

### Phase 1: Agent 私聊建议能力验证

**目的**: 验证 Agent 能实时分析对话，向主人发送仅自己可见的私聊建议，并提供一键采纳功能。

**Step 1.1**: 进入消息列表，创建一个新的 Agent 间对话

- **操作**: 点击底部 Tab "Messages"，进入 MessagesListScreen；点击右上角"+"或"New Chat"进入 NewChatScreen
- **预期**: 看到消息列表为空或已有会话列表；NewChatScreen 显示已配置的 Agent 场景列表（如求职、社交、广告）
- **验证 AC**: `AS-005~func` Agent 自主决定与符合条件的候选 Agent 发起对话

**Step 1.2**: 启动 Agent 自主发起对话流程

- **操作**: 选中一个场景（如"求职"场景），点击某条候选职位，触发 Agent 发送打招呼消息
- **预期**: Agent 自动生成一条打招呼消息并出现在聊天界面中，消息气泡上标注"XX 的 Agent"身份
- **验证 AC**: `AS-005-AC-2` 自主发起对话请求；`US-AGENT-008-AC-2` Agent 消息标识为"XX 的 Agent"

**Step 1.3**: 观察私聊建议面板（侧边栏）

- **操作**: 在聊天进行中，观察屏幕侧边栏或独立建议区域
- **预期**: Agent 实时推送私聊建议，显示话题推荐、风险提示、对方意图分析
- **截图验证**: 侧边栏建议面板截图，显示建议内容
- **验证 AC**: `US-AGENT-009-AC-1` 私聊建议显示在侧边栏或独立区域；`US-AGENT-009-AC-2` 建议包括话题推荐、风险提示、对方意图分析；`AS-008-AC-2` 发送策略建议、风险提示、话题推荐

**Step 1.4**: 一键采纳建议

- **操作**: 点击建议旁的"采纳"或"Use this"按钮
- **预期**: 输入框自动填充建议内容，或直接以用户身份发送该建议消息
- **截图验证**: 输入框填充后截图
- **验证 AC**: `US-AGENT-009-AC-3` 可一键采用建议；`AS-008-AC-3` 提供一键采纳

**Step 1.5**: 验证对方消息的身份显示

- **操作**: 等待对方 Agent 回复，观察消息气泡
- **预期**: 每条消息显示头像和名称；Agent 发送的消息标记为"YY 的 Agent"；用户自己消息显示"我"的身份
- **截图验证**: 对话界面截图，标注用户消息和 Agent 消息的不同标识
- **验证 AC**: `US-AGENT-008-AC-1` 每条消息显示头像和名称；`US-AGENT-008-AC-2` Agent 消息标识为"XX 的 Agent"

### Phase 2: 披露策略与人机模式切换

**目的**: 验证 Agent 在不同信任水平下的信息披露策略，以及用户实时切换"亲自聊"和"让 Agent 聊"的能力。

**Step 2.1**: Agent 执行信息披露策略

- **操作**: Agent 间经过几轮对话后，Agent 根据信任水平决定是否向对方披露主人的详细信息（如薪资底线、具体需求）
- **预期**: Agent 在披露前先向主人汇报（在私聊建议中显示披露提示），或自动按策略执行（每次确认/自动）
- **截图验证**: 披露决策前的确认提示截图
- **验证 AC**: `AS-007-AC-2` 执行披露策略（每次确认/自动）；`AS-007-AC-3` 记录已披露信息；`AS-008-AC-1` 实时分析对方意图和态度

**Step 2.2**: 用户否决即将发生的披露

- **操作**: 若 Agent 显示即将披露的确认提示，用户点击"否决"或"取消"
- **预期**: 披露行为被阻止，Agent 不发送敏感信息；消息状态更新
- **截图验证**: 否决后聊天界面截图，确认敏感信息未被发送
- **验证 AC**: `US-AGENT-014-AC-3` 用户可实时否决即将发生的披露

**Step 2.3**: 切换到"亲自聊"模式

- **操作**: 在四人群聊（或私聊）界面，找到并点击"亲自聊"按钮
- **预期**: 界面状态切换，后续消息以用户本人名义发送；切换状态实时显示在聊天室中
- **截图验证**: 模式切换前后的界面截图，显示状态变化
- **验证 AC**: `US-AGENT-007-AC-1` 点击"亲自聊"按钮后以用户身份发言；`US-AGENT-007-AC-3` 切换状态实时显示在聊天室

**Step 2.4**: 切换回"让 Agent 聊"模式

- **操作**: 点击"让 Agent 聊"按钮
- **预期**: 后续消息改为以 Agent 名义发送；聊天室内实时更新标识
- **截图验证**: Agent 模式下的消息气泡截图，显示"XX 的 Agent"标识
- **验证 AC**: `US-AGENT-007-AC-2` 点击"让 Agent 聊"按钮后 Agent 以自己名义发言

**Step 2.5**: 对方切换时身份标识实时更新

- **操作**: 对方用户也切换模式时，观察消息气泡标识变化
- **预期**: 对方头像/名称旁的 Agent 标识随模式切换实时更新
- **截图验证**: 双方模式不一致时的对话截图
- **验证 AC**: `US-AGENT-008-AC-3` 对方切换时实时更新标识

### Phase 3: 多 Agent 并行对话与协调

**目的**: 验证 Agent 同时与多个候选 Agent 保持独立对话的能力，以及多轮对话中的协商机制。

**Step 3.1**: Agent 同时与多个候选 Agent 保持对话

- **操作**: 打开"求职"场景，让 Agent 同时联系 2 个不同公司的 HR Agent；切换到另一个 Agent 对话窗口
- **预期**: 每个对话窗口独立维护上下文；Agent 能在多个对话中独立发送消息
- **截图验证**: 两个独立对话窗口的截图，并排展示
- **验证 AC**: `AS-005-AC-3` 同时与多个候选保持独立对话；`AS-006-AC-2` 评估是否值得交流

**Step 3.2**: Agent 向主人汇报对话进展

- **操作**: 在 Agent 对话过程中，查看私聊建议面板中的进度汇报
- **预期**: 面板中显示各候选 Agent 的进展摘要（如"已通过初筛，正在协商薪资"）
- **截图验证**: 进度汇报截图
- **验证 AC**: `AS-006-AC-3` 向主人汇报对话进展

**Step 3.3**: 薪资保护验证（Job 场景异常）

- **操作**: 在薪资协商过程中，尝试让 Agent 提交低于主人底线的薪资提案
- **预期**: Agent 拒绝超出底线的提案，显示风险提示
- **截图验证**: Agent 拒绝截图或风险提示截图
- **验证 AC**: `AS-JOB-001-AC-3` 协商薪资保护主人底线

**Step 3.4**: Agent 礼貌结束不适合的对话

- **操作**: Agent 评估某候选 Agent 不匹配后，发起结束对话
- **预期**: Agent 发送礼貌结束语（如"Thank you for your time..."），不会直接消失
- **截图验证**: 结束语消息截图
- **验证 AC**: `US-AGENT-015-AC-2` Agent 礼貌结束对话

### Phase 4: 引荐机制（匹配达标后的真人连接）

**目的**: 验证 Agent 评估匹配度达标后请求主人确认，并将双方主人引入四人群聊。

**Step 4.1**: Agent 评估匹配度并请求引荐

- **操作**: 在 Agent 间对话达到一定轮次后，观察 Agent 是否发送引荐请求
- **预期**: Agent 显示匹配度评分（如"匹配度 85%"），并请求主人确认是否引荐真人
- **截图验证**: 匹配度评分和引荐请求截图
- **验证 AC**: `AS-009-AC-1` 评估匹配度是否达到引荐阈值；`AS-009-AC-3` 主人接受或拒绝；`AS-008-AC-2` 发送策略建议

**Step 4.2**: 主人接受引荐

- **操作**: 点击引荐请求中的"接受"按钮
- **预期**: 界面显示引荐成功提示；同时另一方主人也收到引荐请求（需另一账号配合）
- **截图验证**: 引荐确认截图

**Step 4.3**: 进入四人群聊

- **操作**: 双方都接受引荐后，自动进入四人群聊界面
- **预期**: 四人聊天界面：主人A + A的Agent + 主人B + B的Agent；Agent 在群聊中提供话题建议
- **截图验证**: 四人群聊界面截图，显示四个参与者的头像和名称
- **验证 AC**: `DATE-004-AC-1` 四人群聊界面；`DATE-004-AC-2` Agent 提供交友话题建议

---

### Phase 5: 对话历史与筛选

**目的**: 验证用户能按场景筛选对话记录，并查看历史 Agent 对话详情。

**Step 5.1**: 按场景筛选对话记录

- **操作**: 在 MessagesListScreen 使用场景筛选器（如"求职"、"社交"、"广告"）
- **预期**: 只显示属于该场景的对话记录，其他场景的对话被过滤
- **截图验证**: 筛选后的列表截图
- **验证 AC**: `US-AGENT-005-AC-1` 按场景筛选对话记录

**Step 5.2**: 查看对话详情中的 Agent 信息

- **操作**: 点击某条历史对话，进入 ChatScreen
- **预期**: 显示对话双方 Agent 和匹配度评分
- **截图验证**: 历史对话详情截图，标注 Agent 信息和匹配度
- **验证 AC**: `US-AGENT-005-AC-2` 显示对话双方 Agent 和匹配度评分

### Phase 6: 异常场景与边界处理

**目的**: 验证系统在异常情况下的容错和用户体验。

**Step 6.1**: 异常 - 私聊建议为空

- **操作**: 在极简对话场景下（仅一两句），观察私聊建议面板
- **预期**: 面板显示"信息不足，无法提供建议"或类似提示，不闪退、不崩溃
- **截图验证**: 建议为空的友好提示截图
- **验证 AC**: `US-AGENT-009~edge` 异常：建议为空

**Step 6.2**: 异常 - 建议泄露检查

- **操作**: 在四人群聊中，确认私聊建议不会出现在群聊消息中
- **预期**: 私聊建议面板仅对自己可见，群聊消息流中不出现任何建议内容
- **截图验证**: 对比私聊建议面板和群聊消息流，确认隔离性
- **验证 AC**: `US-AGENT-009~edge` 异常：建议泄露

**Step 6.3**: 异常 - 引荐请求丢失

- **操作**: 网络不稳定或服务器延迟时发送引荐请求，观察重试或超时机制
- **预期**: 显示加载状态或重试按钮，不出现死锁
- **截图验证**: 超时或重试提示截图
- **验证 AC**: `US-AGENT-006~edge` 异常：引荐请求丢失

**Step 6.4**: 异常 - 身份标识错乱

- **操作**: 在快速模式切换场景下，发送多条消息
- **预期**: 每条消息的身份标识正确，不会出现"用户消息显示 Agent 标识"或反向的情况
- **截图验证**: 快速切换后的消息流截图，验证身份一致性
- **验证 AC**: `US-AGENT-008~edge` 异常：身份标识错乱

**Step 6.5**: 异常 - 用户可随时退出

- **操作**: 在对话任意阶段，点击"退出"或关闭对话
- **预期**: 弹出确认对话框，退出后 Agent 礼貌结束对话或显示已离开状态
- **截图验证**: 退出确认截图
- **验证 AC**: `US-AGENT-015~func` 用户在匹配过程中随时退出

**Step 6.6**: 举报功能

- **操作**: 在对话界面点击举报入口（如消息气泡右键或设置菜单）
- **预期**: 弹出举报表单，显示举报原因选项；提交后显示确认
- **截图验证**: 举报表单截图
- **验证 AC**: `US-AGENT-016-AC-1` 可在对话界面举报

**Step 6.7**: 异常 - AI 服务降级

- **操作**: 模拟 AI 服务不可用（如断网或服务降级），发送消息
- **预期**: 显示友好的错误提示"建议暂时不可用"，不崩溃；消息本地缓存，服务恢复后可重发
- **截图验证**: 错误提示截图
- **验证 AC**: `AS-008~edge` 异常：AI 服务降级
