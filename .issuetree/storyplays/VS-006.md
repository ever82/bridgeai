# Story Playbook: 历史照片查询

**Story**: `VS-006` | **Narrative**: 用户查询历史照片。

## 验收旅程概览

本剧本模拟真实用户「李明」使用 VisionShare 搜索本地和云端历史照片的完整旅程。覆盖 L3 自然语言信息模型（文本提炼）、地域过滤系统和本地相册 AI 检索三大核心能力。

**关键能力**:

- 自然语言文本输入 + LLM 智能提炼为结构化查询
- 地点（省市区/距离）/ 时间多维度搜索
- 云端 + 本地相册统一搜索
- 权限引导与隐私保护
- 空结果与异常条件处理

---

## 前置条件

- 设备已安装 VisionShare App（iOS/Android）
- 已登录用户账号（测试账号: `test.vs006@bridgeai.com` / 密码: `Test123456`）
- 测试数据准备:
  - 云端已上传含 EXIF 地点/时间戳的历史照片 20 张（测试地点: 北京/上海/杭州）
  - 本地相册已授权，含 30 张测试照片
  - Agent 已完成部分照片的 AI 标注和向量索引
- 测试地图服务（高德/百度）可用
- 数据库已配置 PostGIS 扩展和地理数据

## 旅程步骤

### Phase 1: 初始探索 —— 文本输入与提炼

**目的**: 验证用户以自然语言描述搜索意图，系统通过 LLM 提炼为结构化查询条件

**Step 1.1: 打开搜索页**

- 操作: 启动 App → 点击底部导航「发现」→ 点击搜索框进入搜索页
- 预期: 搜索页展示搜索历史、热门标签、来源切换（云端/本地/全部）
- 验证 AC:
  - `ISSUE-C002c~c1` — 多段文本输入支持、富文本/纯文本选项
  - `ISSUE-C002c~c3` — 提炼结果展示 UI
  - `ISSUE-VS007~c1` — 权限状态管理（首次进入应有相册权限引导）

**Step 1.2: 输入自然语言查询**

- 操作: 在搜索框输入: "我想找 2023 年夏天在上海拍的照片"
- 预期: 输入框下方实时显示输入字数（字符数限制提示），支持切换富文本/纯文本模式
- 验证 AC:
  - `ISSUE-C002c~c1` — 字符数限制和提示
  - `ISSUE-C002c~c1` — 富文本/纯文本选项

**Step 1.3: LLM 提炼确认**

- 操作: 点击「搜索」或「提炼」按钮
- 预期:
  - 系统展示 LLM 提炼结果卡片: `{地点: 上海, 时间: 2023年夏}`，附带置信度评分（如 92%）
  - 用户可编辑提炼结果字段
  - 提供「确认搜索」和「重新提炼」按钮
- 截图验证: 提炼结果卡片 + 置信度可视化
- 验证 AC:
  - `ISSUE-C002c~c2` — LLM 提炼 L3 到 L2、置信度评分、可编辑的提炼结果
  - `ISSUE-C002c~c3` — 提炼结果展示、编辑和确认流程、拒绝和重新提炼、置信度可视化
  - `ISSUE-C002c~c4` — 提炼历史存储

**Step 1.4: 确认并执行搜索**

- 操作: 点击「确认搜索」
- 预期: 进入结果页，展示符合条件的历史照片网格
- 验证 AC:
  - `VS-006-AC-1` — 支持按地点和时间搜索
  - `VS-006~func` — 用户按地点和时间搜索历史照片

### Phase 3: 地域过滤深化

**目的**: 验证省市区三级联动、距离范围筛选和地理围栏能力

**Step 3.1: 精确地点选择**

- 预期:
  - 地点选择器为三级联动（省→市→区）
  - 地图选点功能可用（点击地图图标展开）
  - 搜索结果仅显示徐汇区范围内标注的照片
- 截图验证: 三级联动选择器 + 地图选点
- 验证 AC:
  - `ISSUE-C003~c2` — 三级联动选择器、地图选点功能

  - `ISSUE-C003~c5` — 地理编码（地址转坐标）

**Step 3.2: 距离范围筛选**

### Phase 5: 本地相册 AI 能力验证

**目的**: 验证本地照片的 AI 分析、索引和隐私保护机制

**Step 5.1: 权限引导与申请**

- 操作: 退出登录 → 重新安装 App → 首次进入搜索页时（尚未授权相册）
- 预期:
  - 弹出权限申请说明对话框（清晰告知权限用途和数据处理方式）
  - 用户可选择「允许」或「暂不」
  - 若拒绝，提供「去设置」入口引导开启权限
- 截图验证: 权限申请弹窗 + 拒绝引导
- 验证 AC:
  - `ISSUE-VS007~c1` — 相册读取权限申请、权限引导说明、权限被拒绝处理
  - `ISSUE-VS007~c5` — 数据处理说明

**Step 5.2: 照片分析与索引**

- 操作: 授予相册权限后，等待系统完成首次批量分析（约30秒）
- 预期:
  - 进度指示器显示「正在分析本地照片... 15/30」
  - 分析完成后，显示索引摘要（已分析 N 张，生成 N 个标签）
  - 后台可继续增量更新，不影响当前操作
- 截图验证: 分析进度弹窗/横幅
- 验证 AC:
  - `ISSUE-VS007~c2` — 本地照片批量处理、端侧AI模型推理、图像特征提取、标签生成、处理进度显示
  - `ISSUE-VS007~c3` — 增量更新机制

**Step 5.3: 隐私模式验证**

- 操作: 设置 → 隐私设置 → 开启「隐私模式」→ 搜索同一关键词
- 预期:
  - 隐私模式开启后，敏感照片（标记为私密）不参与搜索和索引
  - 搜索结果中不出现私密照片，但总计数不透露具体数量（防统计推断）
- 截图验证: 隐私模式开启前后对比（结果一致但无私密照片）
- 验证 AC:
  - `ISSUE-VS007~c5` — 敏感照片排除、隐私模式切换
  - `ISSUE-VS007~c5` — 端侧处理不上传（对比网络请求，无照片上传日志）

**Step 5.4: 索引管理**

- 操作: 设置 → 存储管理 → 查看本地索引信息
- 预期:
  - 显示索引大小（约 X MB）、照片数量、最后更新时间
  - 支持「重建索引」和「清除索引」选项
- 验证 AC:
  - `ISSUE-VS007~c3` — 索引压缩优化、索引备份和恢复、索引版本管理
  - `ISSUE-VS007~c6` — 后台索引更新、内存使用优化

### Phase 7: 异常与边界条件

**目的**: 验证系统在异常场景下的健壮性和用户体验

**Step 7.1: 搜索无结果**

- 操作: 搜索「2010年东京的夜景」
- 预期:
  - 显示空结果状态页（非白屏/无报错崩溃）
  - 空结果页文案: 「未找到符合条件的照片，试试调整关键词或扩大搜索范围」
  - 提供「清除时间筛选」「放宽距离」等快捷操作
- 截图验证: 空结果状态页
- 验证 AC:
  - `VS-006~edge` — 异常：搜索无结果
  - `ISSUE-C003~edge` — 附近无匹配用户（地点无法解析时，提供友好降级）

**Step 7.2: 搜索条件异常**

- 操作:
  - 搜索框输入空白文本，点击搜索
  - 输入非法时间范围（如「2050年」超过当前日期）
  - 输入模糊地点（如「某地」无法解析）
- 预期:
  - 空白输入: 提示「请输入搜索关键词」
  - 非法时间: 提示「时间不能超过当前日期」
  - 无法解析地点: 显示「未找到该地点，请尝试更具体的描述」，并提供手动选择地图的替代方案
- 截图验证: 各异常条件的错误提示
- 验证 AC:
  - `VS-006~edge` — 搜索条件异常处理
  - `ISSUE-C003~edge` — 无法解析地点的友好提示

**Step 7.3: 网络异常**

- 操作: 开启飞行模式 → 尝试搜索云端照片
- 预期:
  - 显示网络错误提示（不崩溃）
  - 本地相册搜索仍可用（离线模式）
  - 网络恢复后自动重试
- 验证 AC:
  - `ISSUE-VS007~c6` — 搜索响应优化
  - 系统容错性（非 AC 但需验证）

**Step 7.4: 权限被拒绝后的降级**

- 操作: 在系统设置中撤销相册权限 → 返回 App 搜索页
- 预期:
  - 搜索页仍可使用云端照片搜索
  - 本地照片入口显示「未授权」，点击可重新引导授权
  - 不因权限缺失导致整体功能不可用（部分访问支持）
- 截图验证: 权限降级后的搜索页状态
- 验证 AC:
  - `ISSUE-VS007~c1` — 部分访问支持(iOS14+)、权限状态管理
