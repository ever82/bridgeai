# Story Playbook: 约会后评分

**Story**: `DATE-005` | **Narrative**: 约会完成后双方评分，影响信用分

## 验收旅程概览

| 维度       | 内容                                                                               |
| ---------- | ---------------------------------------------------------------------------------- |
| 核心场景   | AgentDate 约会场景的完整评分旅程                                                   |
| 覆盖 Issue | C005(信用过滤), CR002(评分系统), CR002a(后端API), CR002b(前端UI), CR002c(业务逻辑) |
| 覆盖 AC    | 44 个（唯一 AC 去重后 38 个）                                                      |
| 旅程类型   | 端到端用户旅程 + 异常路径覆盖                                                      |

## 前置条件

- 双角色已登录 AgentDate 应用（角色A：发起方，角色B：被约方）
- 两个角色已完成一次完整的约会对话（模拟真实交互）
- 数据库中已存在双方的用户画像和信用分
- 评分功能已部署到当前测试环境

## 旅程步骤

### Phase 1: 约会完成与评分触发

**目的**: 验证约会结束后评分界面自动弹出，触发评分流程

**Step 1.1**: 模拟约会结束

- 操作: 通过 API 标记约会状态为「已完成」
- 预期: 系统检测到约会结束，准备推送评分邀请
- 验证 AC: `DATE-005-AC-1` 交流结束后弹出评分界面
- 验证 AC: `DATE-005~func` 约会完成后双方评分，影响信用分

**Step 1.2**: 评分界面自动弹出（A方视角）

- 操作: 角色A进入消息页面，等待评分弹窗出现
- 预期: 约会结束后的聊天界面顶部显示评分邀请卡片，弹出评分表单
- 截图位置: 评分弹窗截图，确认「立即评分」按钮可见
- 验证 AC: `ISSUE-CR002b~c1` 评分输入组件(星级评分+文字评价)

**Step 1.3**: 多维度评分填写

- 操作: 角色A填写四个维度评分（诚信度、礼貌度、响应度、满意度）各5星
- 预期: 每个维度独立滑块/星级选择器，可输入文字评价
- 截图位置: 填写后的评分表单截图
- 验证 AC: `US-AGENT-010-AC-1` 多维度评分（诚信度、礼貌度、响应度、满意度）
- 验证 AC: `ISSUE-CR002b~c1` 移动端评价UI - 评分输入组件

**Step 1.4**: 提交评分

- 操作: 点击「提交评分」按钮
- 预期: 评分成功提交，显示「感谢您的评分」提示，弹窗关闭
- 验证 AC: `ISSUE-CR002a~c2` POST /api/v1/reviews - 提交评价

### Phase 2: 对方评分与信用分联动

**目的**: 验证双向评分机制和信用分实时更新

**Step 2.2**: 角色B填写评分

- 操作: 角色B点击评分入口，填写自己的评分（4星好评）

- 验证 AC: `JOB-005-AC-1` 双向评分影响信用分（跨场景验证）
- 验证 AC: `US-AGENT-010-AC-2` 评分影响信用分

**Step 2.3**: 信用分更新验证

- 操作: 双方各自查看个人资料页面

- 截图位置: 信用分变化前后对比截图

**Step 2.4**: 查看收到的评价

- 操作: 进入「我的」-「评价管理」-「收到的评价」
- 预期: 列表显示刚收到的评价，包含评分星级、文字内容、时间戳

- 验证 AC: `ISSUE-CR002a~c2` GET /api/v1/reviews/received - 获取收到的评价

### Phase 5: 异常场景与反作弊验证

**目的**: 验证恶意评分、重复评分等异常处理

**Step 5.1**: 恶意评分检测

- 操作: 使用测试账号尝试提交包含敏感词的评价（如：政治敏感词、脏话）
- 预期: 提交被拦截，提示「评价内容包含不当词汇」，或提交后被标记待审核
- 验证 AC: `ISSUE-CR002a~c3` 敏感词过滤和AI内容审核

**Step 5.2**: 重复评分拦截

- 操作: 对同一约会尝试再次提交评分
- 预期: 系统拒绝，提示「您已完成评分，无法重复提交」
- 截图位置: 重复评分拦截提示截图
- 验证 AC: `DATE-005~edge` 异常：重复评分
- 验证 AC: `US-AGENT-010~edge` 异常：恶意评分、重复评分

**Step 5.3**: 未完成交易评分拦截

- 操作: 模拟尝试对未完成的约会进行评分
- 预期: 系统拒绝，提示「约会未完成，暂无法评分」
- 验证 AC: `ISSUE-CR002a~c3` 评价真实性验证(需有实际交互)
- 验证 AC: `JOB-005~edge` 异常：未面试就评分

**Step 5.4**: 恶意刷评价检测

- 操作: 同一用户短时间内对多个对象提交评价
- 预期: 系统触发反作弊机制，评价被标记或账户受限
- 验证 AC: `ISSUE-CR002a~c3` 恶意刷评价检测(同一用户多次评价)
