# Story Playbook: 配置招聘方 Agent 画像

**Story**: `JOB-002`
**Narrative**: 招聘方配置 Agent，发布职位需求。

## 验收旅程概览

验证招聘方用户能完整配置 Agent 画像，发布职位需求，设置筛选条件，使 Agent 能精准匹配求职者。

## 前置条件

- 服务已启动
- 数据库已迁移
- 用户已注册并登录
- 用户已创建基础 Agent

## 旅程步骤

### Phase 1: 进入 AgentJob 场景配置

**目的**: 验证招聘方能进入场景并开始画像配置

**Step 1.1**: 选择 AgentJob 场景

- 操作: 在 Agent 列表或创建向导中选择"AgentJob"场景
- 预期: 进入招聘方画像配置页面

**Step 1.2**: 选择角色身份

- 操作: 选择"我是招聘方"
- 预期: 展示招聘方专属配置表单

### Phase 2: 发布职位需求

**目的**: 验证职位发布和需求配置

**Step 2.1**: 填写职位基本信息

- 操作: 填写职位名称、职位描述、工作地点
- 预期: 职位信息正常保存

**Step 2.2**: 设置薪资范围

- 操作: 设置职位薪资范围
- 预期: 薪资范围保存成功，Agent 在筛选时作为参考

**Step 2.3**: 技能要求配置

- 操作: 添加职位要求的技能标签（如"3年+ React"、"本科以上"）
- 预期: 技能要求标签正常展示

**Step 2.4**: 设置筛选条件

- 操作: 配置自动筛选条件（学历要求、经验年限、技能匹配度阈值）
- 预期: 筛选条件保存成功

### Phase 3: 职位管理与预览

**目的**: 验证已发布职位的管理功能

**Step 3.1**: 预览职位卡片

- 操作: 点击"预览职位"
- 预期: 展示求职者视角的职位卡片

**Step 3.2**: 管理已发布职位

- 操作: 在职位列表中查看、编辑、暂停/恢复发布
- 预期: 职位状态正确切换

### Phase 4: 异常场景

**目的**: 验证职位发布的边界情况

**Step 4.1**: 必填字段为空提交

- 操作: 不填写职位名称或描述直接发布
- 预期: 表单校验提示必填项

**Step 4.2**: 重复发布相同职位

- 操作: 发布与已有职位高度相似的职位
- 预期: 系统提示可能重复，用户可确认或修改
