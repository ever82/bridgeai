# Story Playbook: Agent 精准匹配（双向过滤）

**Story**: `AD-003` | **Narrative**: Agent 间匹配用户需求和商家优惠，双向过滤。

## 验收旅程概览

本剧本模拟两个真实用户 -- 消费者"小陈"和商家"王老板" -- 在 AgentAd 场景中完成从画像配置、信息披露策略设置、Agent 精准双向匹配、到匹配结果浏览与异常处理的完整旅程。旅程以「消费者寻找装修优惠 + 商家推广装修活动」为主线，贯穿 L1/L2/L3 三层信息模型和披露控制机制，自然覆盖全部 67 个唯一 AC。

**角色设定**:

- **小陈** (消费者): 想找装修公司优惠，首次使用 AgentAd
- **王老板** (商家): 经营装修公司，有 5 月满减活动，使用 AgentAd 推广

## 前置条件

1. 已安装 BridgeAI mobile app (iOS/Android)
2. 已有 BridgeAI 账号并完成基础认证 (两个测试账号)
3. 数据库已迁移并包含测试用户数据
4. 服务端运行: `pnpm dev` (apps/server, apps/mobile)
5. 消费者测试账号: `test+consumer@bridgeai.com` / `password123`
6. 商家测试账号: `test+merchant@bridgeai.com` / `password123`
7. 后台已存在其他场景的 Agent 数据 (用于跨场景验证)
