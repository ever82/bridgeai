# Story Playbook: AI 智能检索本地相册

**Story**: `VS-003`
**Narrative**: 系统智能扫描本地相册，推荐匹配照片。

## 验收旅程概览

作为 VisionShare 接单者，从发现任务、查看任务详情开始，经历 Agent 智能扫描本地相册并推荐匹配照片、人工确认推荐结果、拍摄新照片、隐私脱敏处理、照片上传交付、Agent 私聊建议查看、匹配确认与引荐的完整旅程，自然覆盖 L2 结构化信息模型、Agent 对话生成服务（意图分析/协商/记忆/人格/切换）、相机拍照与相册导入全链路的所有关键 AC。

## 前置条件

- 服务已启动：`pnpm dev`
- 数据库已迁移：`pnpm db:migrate && pnpm db:seed`
- 测试用户账号已创建（信用分 85，积分 200，可拍照可接单）
- 手机相册中有不同地点/时间的测试照片（至少包含：深圳、广州风景照片各 3 张）
- 手机模拟器已配置或真机已连接
- 相机权限和相册读取权限均已授予

## 旅程步骤

---

### Phase 1: 任务发现与筛选

**目的**: 接单者在地图上发现附近任务，验证地域过滤和信用过滤条件正确生效。

---

**Step 1.1**: 进入 VisionShare 并打开附近任务地图

- 操作: 启动 App，底部导航点击 VisionShare 图标，切换到「附近任务」Tab，进入地图视图
- 预期:
  - 地图居中显示当前位置（蓝色定位点）
  - 地图上显示任务标记（蓝色圆形标记）
  - 底部有列表视图切换按钮
- 验证 AC: `ISSUE-VS002~c1` 地图界面（任务分布展示、当前位置定位、任务标记）

---

**Step 1.2**: 应用过滤条件（地域 + 信用）

- 操作: 点击地图右上角筛选按钮，设置：
  - 距离：「5km」
  - 预算：「40 积分以上」
  - 排序：「距离优先」
- 预期:
  - 地图缩放，标记减少，仅显示符合条件任务
  - 列表视图同步刷新
  - 每个任务标记旁显示距离（如「1.2km」）
- 验证 AC: `VS-002-AC-1` 只显示符合地域和信用过滤条件的任务；`ISSUE-VS002~c2` 筛选功能（距离、预算、排序）；`AS-005-AC-1` 根据过滤规则筛选候选；`AS-VS-001-AC-3` 筛选信用高、距离近的候选

---

**Step 1.3**: 查看任务详情

- 操作: 点击地图上任务标记，或在列表中点击任务项
- 预期:
  - 弹出任务详情卡片，显示：任务标题、地点、时间、预算、发布者信用分
  - 显示「距离：X.Xkm，预计到达 Y 分钟」
  - 显示任务状态标签：「进行中」「匹配中」「已发布」
- 验证 AC: `ISSUE-VS002~c3` 详情展示（任务信息、信用分、距离）；`AS-009-AC-2` 向主人展示对方信息和信用分；`US-AGENT-013-AC-1` Agent 展示对方公开需求概要

---

**Step 1.4**: 接单资格检查（前置条件）

- 操作: 在任务详情页点击「接单」按钮
- 预期（条件满足）:
  - 弹出确认弹窗：「确认接单？完成后将获得 XX 积分」
  - 点击「确认」后状态变为「进行中」
  - 底部导航出现「拍照」快捷入口
- 预期（信用不足）:
  - 显示「您的信用分（75）低于任务要求（80），无法接单」
- 预期（位置异常）:
  - 显示「无法获取您的位置，请开启定位权限」
- 验证 AC: `ISSUE-VS002~c4` 接单流程（资格检查、状态变更）；`ISSUE-C005~c4` 不足处理（信用不足原因、提升建议）；`VS-002-AC-2` 点击接单后进入拍照界面；`VS-002~edge` 异常：位置异常、信用过滤绕过；`AS-VS-002~func` Agent 判断主人是否适合响应附近视觉任务

### Phase 2: Agent 智能扫描本地相册

**目的**: 接单成功后，Agent 自动扫描本地相册，根据任务需求（L1 地点 + L2 拍照类型）推荐匹配照片，展示智能推荐结果。

---

**Step 2.1**: Agent 激活并读取相册

- 操作: 从「进行中」任务点击「查看推荐」，或系统自动推送推荐通知
- 预期:
  - 界面显示「Agent 正在分析你的相册...」
  - 完成后显示推荐照片列表，按匹配度排序
  - 每张照片显示匹配度百分比（如「匹配度 92%」）和推荐理由
  - 推荐理由示例：「地点：深圳湾公园 | 类型：风景」
- 验证 AC: `VS-003~func` 系统扫描本地相册并推荐匹配照片；`VS-003-AC-2` 符合条件的照片推荐给用户确认；`ISSUE-C002b~c1` 结构化信息定义（场景 L2 Schema：地点 enum、时间 range、拍照类型 multi_select）；`ISSUE-C002b~c3` L2 存储（JSON 字段存储、GIN 索引）；`AS-002~func` Agent 解析主人自然语言输入，提炼成结构化需求；`AS-003~func` Agent 理解主人供给能力，记录为供给档案；`ISSUE-AI004~c4` 记忆系统（短期对话记忆、关键信息提取）

---

**Step 2.2**: 查看智能相册（分类视图）

- 操作: 在推荐列表页点击「智能相册」Tab，进入分类视图
- 预期:
  - 显示按地点分类的相册组：「深圳」「广州」「其他」
  - 每组显示照片数量和代表性缩略图
  - 显示按时间分布的视图切换选项
- 验证 AC: `ISSUE-VS003~c6` 导入功能（本地相册访问、多选照片导入、导入照片预览）；`ISSUE-C002b~c2` 表单渲染（根据 Schema 动态生成/显示）；`ISSUE-C002b~c4` L2 配置界面（实时预览、自动保存草稿）

---

**Step 2.3**: 搜索相册

- 操作: 在相册页顶部搜索框，输入「深圳湾」
- 预期:
  - 实时显示搜索结果，按相关性排序
  - 结果中照片显示拍摄地点标签和时间
  - 显示照片数量统计（如「找到 12 张照片」）
- 验证 AC: `ISSUE-VS003~c6` 导入功能（照片来源标记）；`ISSUE-C002b~c3` L2 存储（GEX 索引支持复杂查询）

---

**Step 2.4**: 确认推荐照片

- 操作: 在推荐列表勾选 3 张推荐照片，点击「确认使用」
- 预期:
  - 弹出确认：「已选择 3 张照片，将用于提交」
  - 选中照片显示绿色勾选标记
  - 页面底部显示「已选 3 张 | 待上传 3 张」
  - 完成后自动进入隐私编辑流程
- 验证 AC: `VS-003-AC-2` 符合条件的照片推荐给用户确认；`ISSUE-VS003~c6` 导入功能（照片选择/取消）；`ISSUE-C002b~c2` 表单渲染（字段联动）；`ISSUE-C002b~c5` 数据验证（Schema 验证、验证错误提示）

---

**Step 2.5**: Agent 实时私聊建议（推荐阶段）

- 操作: 在推荐列表页面，同时查看聊天面板
- 预期:
  - Agent 发送私聊消息（仅自己可见）：「根据任务需求，我推荐优先选择第 2 张和第 5 张，构图更接近发布者要求」
  - 侧边栏显示私聊建议区域
  - 建议包括匹配度分析和理由
- 验证 AC: `US-AGENT-009-AC-1` 私聊建议显示在侧边栏或独立区域；`US-AGENT-009-AC-2` 建议包括话题推荐、风险提示、对方意图分析；`US-AGENT-009~func` Agent 实时向用户发送私聊建议，仅自己可见；`AS-008-AC-2` 发送策略建议、风险提示、话题推荐；`AS-008~func` Agent 实时分析对话，向主人发送私下建议；`AS-008~edge` 异常：建议为空、AI 服务降级；`US-AGENT-009~edge` 异常：建议泄露、建议为空

---

**Step 2.6**: 无匹配照片异常

- 操作: 切换到一个与相册照片地点完全不同的任务（如：任务要求广州，你只有深圳照片），重新触发扫描
- 预期:
  - 显示「未找到符合条件的照片」
  - Agent 推送建议：「相册中没有匹配照片，建议拍照新照片」
  - 页面提供「去拍照」快捷入口
- 验证 AC: `AS-VS-002~edge` 异常：无匹配照片、脱敏失败；`ISSUE-AI004~c4` 记忆系统（无结果处理）

### Phase 4: 照片编辑与隐私脱敏

**目的**: 对选中照片进行裁剪、滤镜等编辑，并进行隐私脱敏处理。

---

**Step 4.1**: 进入照片编辑界面

- 操作: 从草稿列表点击任意一张照片，进入编辑页
- 预期:
  - 全屏显示照片
  - 底部显示编辑工具栏：裁剪、旋转、滤镜、调色
  - 顶部显示任务要求辅助线（如构图参考线）
- 验证 AC: `ISSUE-VS003~c2` 编辑功能（裁剪和旋转、滤镜和亮度调整、标记和注释）

---

**Step 4.2**: 编辑操作

- 操作:
  - 点击裁剪，拖动边框调整比例（选择 4:3）
  - 点击旋转 90 度
  - 选择一个滤镜（暖色调）
  - 调节亮度 +10
- 预期:
  - 裁剪框实时预览
  - 旋转后照片角度更新
  - 滤镜和亮度实时应用
  - 点击「保存」应用所有编辑
- 验证 AC: `ISSUE-VS003~c2` 编辑功能（裁剪和旋转、滤镜和亮度调整）；`ISSUE-VS003~c2` 照片选择/取消

---

**Step 4.3**: 隐私脱敏处理

- 操作: 在编辑页点击「脱敏」Tab，进入隐私编辑器
- 预期:
  - 照片上叠加脱敏工具（涂抹模糊、马赛克、文字遮盖）
  - 智能识别人脸，显示人脸区域框
  - 用户可选择脱敏方式处理人脸
  - 显示隐私等级：「已脱敏」
- 验证 AC: `AS-VS-002-AC-3` 处理隐私脱敏并上传；`ISSUE-VS003~c3` 安全检测（人脸识别和隐私检测）；`ISSUE-VS003~c2` 标记和注释（脱敏工具）

**Step 4.4**: 隐私脱敏失败异常

  - 脱敏失败时仍可手动处理后继续
- 验证 AC: `AS-VS-002~edge` 异常：无匹配照片、脱敏失败；`ISSUE-VS003~c3` 安全检测（隐私检测）

### Phase 6: Agent 对话生成服务验证

**目的**: 验证 Agent 对话引擎的完整能力——意图分析、私聊建议、协商机制、人格一致性、记忆系统、切换机制。

---

**Step 6.1**: Agent 分析对方意图并发送策略建议

- 操作: 进入「我的 Agent > 对话」Tab，查看与当前任务发布者的对话

## 环境启动

```bash
# 1. 安装依赖（如果尚未安装）
pnpm install

# 2. 数据库迁移
pnpm db:migrate
pnpm db:seed    # 创建测试用户

# 3. 启动开发服务器（mobile + server）
pnpm dev

# 4. 运行 UI 测试（非必须，用于截图验证）
pnpm test:e2e:web --check

# 5. 运行集成测试（如果已实现）
pnpm test:e2e:check
```
