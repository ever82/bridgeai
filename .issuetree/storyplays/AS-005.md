# Story Playbook: 我主动与其他 Agent 对话

**Story**: `AS-005` | **Narrative**: 拿到候选列表后自主决定与符合条件的候选 Agent 发起对话。

## 验收旅程概览

本剧本以一对真实用户（Alice 与 Bob）和他们各自的 Agent（Agent-Alice、Agent-Bob）作为主线人物，模拟 Agent-Alice 从获取候选列表 → 自主评估筛选 → 主动发起对话 → 多候选并发对话 → 实时分析对方意图 → 发送私聊建议 → 请求引荐 → 主动退出的完整有机旅程。

同时横向覆盖：

- **AgentJob 场景**（Agent-Alice 同时与多 HR Agent 协商薪资）
- **AgentAd 场景**（Agent-Alice 与商家 Agent 谈判优惠）
- **VisionShare 场景**（Agent-Alice 扫描相册推荐匹配照片）
- **底层基础设施**（Socket.io 通信、房间管理、AI 对话引擎、人格系统、记忆系统）

| 阶段                | 核心操作                                 | 覆盖 AC 数 |
| ------------------- | ---------------------------------------- | ---------- |
| Phase 1: 候选就绪   | Agent 读取画像、学习规则、获取候选列表   | 14         |
| Phase 2: 自主发起   | Agent 评估候选、主动建立对话、多候选并发 | 12         |
| Phase 3: 实时分析   | Agent 分析意图、发送私聊建议、汇报进展   | 8          |
| Phase 4: 引荐确认   | 匹配度达标后请求主人确认引荐             | 6          |
| Phase 5: 横向场景   | AgentJob/AgentAd/VisionShare 横向复用    | 8          |
| Phase 6: 异常处理   | 拒绝对话、并发过载、建议为空等边界       | 10         |
| Phase 7: 退出与记忆 | 礼貌退出、频繁退出限制、Agent 记忆学习   | 8          |

**总 AC 数**: 46 | **目标覆盖**: 37+ (80%+ 有机覆盖)
