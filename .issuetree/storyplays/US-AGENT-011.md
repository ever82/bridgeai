# Story Playbook: 配置需求信息（分公开和详细）

**Story**: `US-AGENT-011`
**Narrative**: 用户为每个场景分别配置公开和详细两级需求信息

## 验收旅程概览

用户"李明"完成从账号注册到 Agent 创建、场景需求配置的完整旅程：先完成 L1 基础画像，再为选定的场景（AgentJob + AgentDate）分别配置公开的需求信息和仅自己可见的详细信息，过程中验证场景切换、模板应用、披露策略切换等异常行为。

- 服务已启动：`pnpm dev`（server + mobile）
- 数据库已迁移：`pnpm db:migrate`

- 测试账号：手机号 `13800001111`，验证码 `123456`

### Phase 1: L1 基础画像 —— Agent 的主人是谁

**目的**: 建立主人基础画像，为 Agent 提供背景知识；验证 Agent 创建后能读取主人画像并生成本场景性格（`AS-001~func` + `AS-001-AC-1`）。

**Step 1.1**: 进入编辑个人资料页面

- 操作: 点击底部 Tab「我的」→「编辑资料」
- 预期: 页面加载 L1ProfileForm，显示「基本信息」和「职业与教育」两个 Section，进度条为 0%
- 验证 AC: `AS-001-AC-1` 读取主人的基础画像和场景配置 —— 验证页面能加载主人的已有 L1 数据（首次为空白表单）

**Step 1.2**: 填写基础信息

- 操作: 依次填写年龄（选择「25-34岁」）、性别（选择「男」）、省份+城市（广东+深圳）、职业（输入「前端工程师」）、学历（选择「本科」）
- 预期: 进度条随填写的必填项增加；字符计数实时更新；表单自动草稿保存（1500ms 防抖）
- 验证 AC: `AS-001~func` Agent 被创建后能读取主人画像 —— 验证 L1 数据在本地草稿存储中正确持久化

**Step 1.3**: 保存并验证 Agent 能读取画像

- 操作: 点击「保存」按钮，弹出「成功」Toast
- 预期: 页面返回，API 调用成功（HTTP 200）；数据库 `user_profiles.l1` 字段有值
- 截图点: Toast 弹窗 + 进度条 100% 绿色状态
