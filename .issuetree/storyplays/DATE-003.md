# Story Playbook: Agent间对话匹配

**Story**: `DATE-003` | **Narrative**: 双方 Agent 先进行对话，匹配度高后请求引荐。

---

## 验收旅程概览

```
用户注册/画像 → Agent 学习过滤规则 → 发现候选（地域+信用过滤）
  → Agent 间隔离对话 → 对话质量监控 → 生成摘要报告 → 用户决策
```

**核心场景**：Alice（上海，信用780）和 Bob（苏州，信用820）通过各自 Agent 完成一次完整的对话匹配体验。

**验收范围**：

- 地域过滤系统（ISSUE-C003）
- 信用过滤系统（ISSUE-C005）
- Agent间对话匹配核心机制（ISSUE-DATE003）

---

## 旅程步骤

### Phase 1: 用户画像与过滤规则初始化

**目的**: 验证 Agent 能读取主人画像并学习场景过滤规则

**Step 1.1**: Alice 完成 AgentDate 场景注册并绑定位置

- 操作：Alice 打开 AgentDate tab，完成手机号验证 → 设置昵称/头像/简介 → 开启定位权限
- 预期：Agent 成功获取 Alice 的经纬度（上海：121.47, 31.23），地理位置解析显示"上海市浦东新区"
- 截图点：个人资料页显示城市标签 + 地图定位标记
- 验证 AC：`ISSUE-C003~c5` 位置服务-地理编码集成、`ISSUE-C003~c6` Agent位置数据存储、`AS-001~func` Agent读取主人画像

**Step 1.2**: Alice 设置地域过滤偏好

- 操作：Alice 进入"发现设置" → 选择"只看附近的人" → 滑动距离滑块设为 50km → 选择省份"上海市"+"江苏省"
- 预期：过滤条件保存成功，设置页显示"上海市 + 江苏省，50km 范围"
- 截图点：过滤设置面板 + 地图围栏可视化（50km 半径圈）
- 验证 AC：`ISSUE-C003~c2` 位置选择UI（三级联动/地图选点）、`ISSUE-C003~c3` 距离筛选滑块、`VS-001-AC-3` 过滤条件设置、`AS-005-AC-1` 根据过滤规则筛选候选

**Step 1.3**: Alice 设置信用过滤偏好

- 操作：Alice 在"发现设置" → 信用筛选区域 → 拖动最低信用分滑块至 700 分 → 选择信用等级"良好及以上"
- 预期：信用门槛保存，显示"信用分 ≥ 700，信用等级：良好/优秀"
- 截图点：信用筛选面板（含信用等级快速选择标签）
- 验证 AC：`ISSUE-C005~c1` 信用分范围筛选、`ISSUE-C005~c3` 信用分滑块UI

**Step 1.4**: Alice 的 Agent 学习过滤规则

- 操作：Alice 查看 Agent 状态页 → 确认 Agent 已加载"AgentDate 场景规则"（地域+信用双过滤）
- 预期：Agent 状态显示"规则已学习：过滤条件 2 条"，可展开查看具体规则内容
- 截图点：Agent 详情页规则列表（地域+信用规则项）
- 验证 AC：`AS-001-AC-2` 学习本场景过滤规则、`AS-001~func` Agent读取画像/学习规则/生成性格

### Phase 2: 候选发现与过滤验证

**目的**: 验证地域过滤和信用过滤正确运作，Agent 扫描并推荐符合条件的对象

**Step 2.1**: Alice 启动 Agent 推荐

- 操作：Alice 点击"开始发现" → Agent 进入后台工作状态 → 页面显示"正在为你寻找..."
- 预期：Agent 实时日志显示正在读取候选库、逐条应用地域过滤（距离 ≤ 50km）、应用信用过滤（信用分 ≥ 700）
- 截图点：Agent 工作状态页 + 后台日志面板（显示过滤步骤）
- 验证 AC：`DATE-002-AC-1` Agent只扫描符合过滤条件的用户、`AS-DATE-001~func` 在符合过滤条件的用户中寻找高匹配度对象

**Step 2.2**: 验证过滤结果正确性

- 操作：Alice 查看推荐列表 → 列表中仅显示距离 50km 内且信用分 ≥ 700 的用户
- 干扰项：附近 80km 处的用户、信用分 650 的用户均不在列表中
- 截图点：推荐列表（含每条结果的"距离 Xkm"和"信用等级"标签）
- 验证 AC：`ISSUE-C003~c3` 基于经纬度距离计算、`ISSUE-C005~c1` 信用分范围筛选、`AS-DATE-001-AC-1` 应用地域和属性过滤

**Step 2.3**: 匹配度排序验证

- 操作：Alice 查看推荐列表排序 → 确认按匹配度分数降序排列
- 预期：排名靠前的候选与 Alice 画像匹配度最高（兴趣/年龄/距离综合分）
- 截图点：推荐列表 + 匹配度分数气泡
- 验证 AC：`AS-004-AC-3` 计算匹配度分数并排序推荐、`AS-004~func` Agent批量读取候选信息计算匹配度

**Step 2.4**: Agent 选择 Bob 并请求对话许可

- 操作：Alice 的 Agent 选定 Bob（苏州，信用 820，匹配度 89%）→ 页面弹出"Agent 想和 Bob 的 Agent 对话，是否同意？"
- 预期：弹窗显示 Bob 的公开画像摘要 + Agent 匹配理由（"共同话题：摄影、旅行"）
- 截图点：对话许可弹窗
- 验证 AC：`DATE-003~func` 双方Agent先对话匹配度高后请求引荐、`AS-DATE-002~func` Agent与对方Agent聊天评估性格匹配度

### Phase 3: Agent 间隔离对话

**目的**: 验证双方 Agent 在隔离环境中进行多轮对话，对话状态正确管理

**Step 3.1**: 对话房间创建与隔离验证

- 操作：Alice 同意后 → 系统创建双人 Agent 对话房间 → Alice 和 Bob 均收到"Agent 对话进行中"通知
- 预期：
  - Alice 端显示"Agent 正在与 Bob 的 Agent 对话中"（不暴露 Bob 实时内容）
  - Bob 端同理
  - 后台显示对话状态为"进行中"，双方 Agent 隔离（彼此不暴露主人信息）
- 截图点：Alice 端对话状态页 + Bob 端对话状态页（对称视图）+ 后台房间管理面板
- 验证 AC：`ISSUE-DATE003~c1` 隔离机制-双人房间创建/状态管理/超时控制/用户不可见隔离层

**Step 3.2**: 多轮对话生成

- 操作：对话自动进行 → Alice 端实时显示 Agent 对话轮次进度（Round 1/5, 2/5...）
- 预期：
  - Agent 基于双方交友画像选择话题（从浅入深：兴趣 → 价值观 → 生活方式）
  - 话题深度逐步增加，Agent 维持人设一致性
  - 多轮对话上下文连贯，无矛盾（不重复问已问问题）
- 截图点：对话轮次进度条 + 各轮话题变化截图（至少 3 个不同话题阶段）
- 验证 AC：`DATE-003-AC-1` Agent间多轮对话、`ISSUE-DATE003~c2` 对话生成（画像策略/话题引导/人设一致性/上下文记忆）

**Step 3.3**: 对话质量实时监控

- 操作：观测对话进行过程中的质量指标面板（后台或调试模式）
- 预期：系统实时计算并显示：
  - 对话流畅度分数（>60 正常，<60 触发调整）
  - 话题深度评分
  - 双方参与度（确保不是单方主导）
  - 信息获取有效性
- 截图点：质量监控仪表盘（多指标同时可见）
- 验证 AC：`ISSUE-DATE003~c3` 质量评估（流畅度/深度/参与度/有效性）

**Step 3.4**: 对话超时与轮次限制

- 操作：等待对话达到单轮超时或总轮次上限
- 预期：对话正常结束，不因超时异常中断；达到轮次限制后自动转入报告生成
- 截图点：对话结束通知（含正常结束原因说明）
- 验证 AC：`ISSUE-DATE003~c1` 超时控制和轮次限制配置

### Phase 4: 报告生成与展示

**目的**: 验证对话结束后生成完整的摘要和匹配评估报告

**Step 4.1**: 对话摘要生成

- 操作：对话结束后 → 页面跳转至"对话报告" → 显示摘要内容
- 预期：
  - 关键信息提取：Bob 的 Agent 透露 Bob 喜欢摄影、在互联网行业工作、周末常去户外
  - 共同话题亮点：摄影（双方均喜欢旅行摄影）、宠物（Bob 养猫）
  - 潜在问题提示：Bob 工作较忙，周末时间可能不固定（轻微提醒，非强提示）
- 截图点：对话摘要卡片（关键信息 + 共同话题高亮）
- 验证 AC：`ISSUE-DATE003~c4` 报告生成（摘要/共同话题/潜在问题）

**Step 4.2**: 匹配度重新评估报告

- 操作：Alice 查看匹配评估报告 → 对话后匹配度（基于真实对话）对比对话前匹配度
- 预期：
  - 对话前匹配度：89% → 对话后匹配度：92%（因为发现共同摄影爱好）
  - 报告清晰标注分数变化原因
- 截图点：匹配评估报告（含前后对比 + 原因说明）
- 验证 AC：`ISSUE-DATE003~c4` 匹配度重新评估（基于对话）、`AS-DATE-002~func` Agent与对方Agent聊天评估性格匹配度

**Step 4.3**: Agent 请求引荐

- 操作：匹配度达标（>85%）→ Bob 的 Agent 发出引荐请求 → Alice 收到"Bob 想与你交流"通知
- 预期：引荐请求弹窗包含 Bob 公开资料 + 对话亮点 + 匹配理由
- 截图点：引荐请求通知 + 详情弹窗
- 验证 AC：`DATE-003-AC-2` 匹配度达标后请求引荐、`DATE-003~func` 高匹配度后请求引荐

### Phase 5: 用户决策与异常处理

**目的**: 验证用户在移动端查看报告并做出决策，同时覆盖异常流程

**Step 5.1**: 移动端报告查看

- 操作：Alice 在手机端打开 AgentDate → 进入"对话报告" tab → 查看完整报告
- 预期：
  - Agent 对话进行中状态提示（若还在进行）
  - 对话完成通知（推送消息）
  - 报告展示：摘要 + 匹配评估 + 共同话题 + 建议
  - 操作按钮：同意引荐 / 婉拒 / 再聊一轮
- 截图点：手机端报告页（关键信息完整显示，操作按钮可见）
- 验证 AC：`ISSUE-DATE003~c5` 移动端UI（状态提示/通知/摘要/决策操作）

**Step 5.2**: Alice 同意引荐

- 操作：Alice 点击"同意引荐" → 确认决策 → 页面显示"已通知 Bob，你们可以开始交流了"
- 预期：Bob 收到通知，双方建立连接，聊天窗口开启
- 截图点：决策确认页 + 双方聊天窗口建立截图
- 验证 AC：`DATE-003-AC-2` 匹配度达标后请求引荐（用户同意链路）

**Step 5.3**: 异常处理 - 过滤后无候选

- 操作：创建极端过滤条件（信用分 ≥ 900，距离 ≤ 1km）→ 启动 Agent 发现
- 预期：
  - Agent 扫描完成，0 候选
  - Alice 收到提示"附近暂未找到符合条件的对象，试试放宽条件？"
  - 显示调整建议（降低信用门槛或扩大距离）
- 截图点：无候选结果页 + 调整建议
- 验证 AC：`ISSUE-C005~c4` 信用不足处理/替代推荐、`AS-DATE-001~edge` 过滤后无候选异常

**Step 5.4**: 异常处理 - 对话质量低

- 操作：使用两个画像差异极大的用户（如 A：宅家游戏，B：户外极限运动）触发低质量对话
- 预期：
  - 系统检测到对话质量分数持续偏低（<40）
  - Agent 自动调整话题策略（从共同话题切入而非直接对比）
  - 若调整无效，对话提前终止并生成质量报告
  - 用户收到"对话匹配度不足，建议重新匹配"通知
- 截图点：质量警告通知 + 提前终止报告
- 验证 AC：`ISSUE-DATE003~c3` 实时质量分数低质量时调整策略、`AS-004~edge` 匹配算法出错异常

**Step 5.5**: 异常处理 - Agent 对话卡住

- 操作：模拟 Agent 对话卡在某一轮（如双方 Agent 同时等待对方回复超过 30 秒）
- 预期：
  - 超时控制触发，对话房间自动结束
  - 生成部分报告（已进行的轮次摘要）
  - 用户收到"对话意外中断，已生成部分报告"通知
  - 提供"重试对话"选项
- 截图点：中断通知 + 部分报告页
- 验证 AC：`DATE-003~edge` Agent对话卡住/引荐请求未到达异常

**Step 5.6**: 安全机制验证

- 操作：对话过程中触发敏感话题（如涉及金钱/政治/不当内容）→ 观测系统响应
- 预期：
  - 敏感话题检测触发，内容被拦截或替换为安全回复
  - 对话异常中断（严重情况下）
  - 用户可手动终止对话
  - 有争议内容时可上报
- 截图点：敏感内容拦截提示 + 手动终止按钮
- 验证 AC：`ISSUE-DATE003~c6` 安全机制（敏感检测/不当拦截/用户终止/争议上报）

### Phase 6: 跨场景扩展验证（VisionShare / AgentJob / AgentAd）

**目的**: 验证地域和信用过滤系统在其他 Agent 场景中统一运作

**Step 6.1**: VisionShare 场景 - 发布视觉需求含地域过滤

- 操作：用户在 VisionShare 发布"需要附近摄影师帮我拍产品照" → 自动解析地点为当前位置 → 发布后仅附近用户可见
- 截图点：VisionShare 发布页（含地点解析确认）+ 需求详情页（仅附近用户推送通知）
- 验证 AC：`VS-001-AC-3` VisionShare设置过滤条件、`VS-001~func` 系统解析地点推送给附近用户、`VS-001~edge` 无法解析地点异常

**Step 6.2**: VisionShare 场景 - 承接任务时地域+信用+相册匹配过滤

- 操作：摄影师用户查看"附近任务" → 系统检查摄影师位置与任务位置匹配度 + 摄影师相册风格与任务需求匹配度 → 列表中仅显示位置和相册均匹配 + 信用达标的任务
- 预期：任务列表每项标注"位置匹配"和"相册风格匹配"标签，不符合的任务不可见
- 截图点：任务列表（含距离标签 + 信用等级 + 相册匹配标签）
- 验证 AC：`VS-002-AC-1` 只显示符合地域和信用过滤条件的任务、`AS-VS-002-AC-1` 检查位置和相册匹配、`VS-002~func` 用户根据地域和信用过滤条件查看并接附近任务、`AS-VS-001-AC-3` 筛选信用高距离近的候选

**Step 6.3**: VisionShare 历史照片地点搜索

- 操作：用户输入"上海外滩" → 系统返回该地点及附近拍摄的历史照片（按时间排序）
- 截图点：地点搜索结果页 + 照片列表
- 验证 AC：`VS-006-AC-1` 支持按地点和时间搜索、`VS-006~func` 用户按地点和时间搜索历史照片

**Step 6.4**: AgentJob 场景 - 职位过滤

- 操作：求职者设置"工作地点：上海"+"信用要求：良好" → Agent 推送的职位列表满足地域+信用双重过滤
- 截图点：职位推荐列表（含地点标签 + 企业信用标签）
- 验证 AC：`AS-JOB-001-AC-1` AgentJob应用过滤条件、`AS-JOB-001~func` 求职者Agent匹配职位

**Step 6.5**: AgentAd 场景 - 广告过滤

- 操作：用户设置广告偏好（地域+信用过滤）→ 系统仅推送高匹配度且符合过滤条件的广告
- 截图点：广告推送列表（含过滤条件匹配标签）
- 验证 AC：`AD-003-AC-1` 只推送高匹配度且符合过滤条件的广告、`AD-003~func` Agent间匹配用户需求和商家优惠

**Step 6.6**: 用户紧急终止对话

- 操作：Alice 在对话进行中点击"终止对话"按钮
- 预期：对话立即中断，双方 Agent 退出隔离房间，Alice 收到确认提示
- 截图点：终止确认弹窗 + 终止后状态页
- 验证 AC：`ISSUE-DATE003~c6` 用户紧急终止对话

## 环境启动

```bash
# 1. 安装依赖
pnpm install

# 2. 数据库迁移
pnpm db:migrate
pnpm db:seed   # 创建种子用户：Alice、Bob、干扰用户若干

# 3. 启动开发服务
pnpm dev

# 4. 运行 e2e 测试（如有对应测试）
pnpm test:e2e

# 5. 打开移动端
# iOS: ios simulator → open AgentDate tab
# Android: android emulator → open AgentDate tab
# Web: http://localhost:3000/m/agentdate

# 6. 截图输出目录
ls e2e_screenshots/
```

## 备注

- **AC 去重**：本 playbook 中每个 AC slug 只列出一次，但同一 AC 可能被多个步骤覆盖（如 `AS-001-AC-2` 在 Step 1.4 验证，也在其他步骤隐含使用）
- **截图策略**：每个 phase 至少 2 张关键截图；异常场景需截图异常状态和恢复路径
- **并行测试**：Step 6（跨场景验证）可与 Step 3-5 并行执行，分别在 VisionShare/AgentJob/AgentAd 标签页操作
- **调试模式**：建议开启 Agent 工作日志（`AGENT_DEBUG=true`）以便观测过滤和匹配过程
