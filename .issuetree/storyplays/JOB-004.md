# Story Playbook: 四人群聊薪资协商

**Story**: `JOB-004` | **Narrative**: 双方 Agent 和用户在四人群聊中协商薪资。

## 验收旅程概览

| 维度     | 内容                                                             |
| -------- | ---------------------------------------------------------------- |
| 角色     | 求职者（候选人）、招聘方（HR）、双方 AI Agent                    |
| 场景     | AgentJob 求职平台 — 匹配后进入四人群聊进行薪资协商               |
| 路径长度 | 6 个 Phase，18 个 Step                                           |
| 覆盖 AC  | 25 unique AC / 26 JSON 条目 (100%，`JOB-004~func` 跨 Issue 重复) |

---

## 前置条件

1. **环境**: `pnpm dev` 启动后端 + 前端，服务地址 `http://localhost:3000`
2. **用户**: 两个已注册账号 — 求职者（candidate@example.com）和招聘方（recruiter@company.com），均已配置 AI Agent
3. **数据**: 招聘方已发布职位 JD，薪资范围 15K–25K；求职者已投递并通过初步筛选
4. **Socket.io 连接**: 两端均已建立有效 WebSocket 连接（token 认证通过）
5. **进入条件**: 双方均接受匹配意向，进入薪资协商流程

---

## 旅程步骤

### Phase 1: 匹配确认与群聊进入

**目的**: 验证双方接受匹配后，系统自动创建四人群聊房间，双方用户和 Agent 成功加入

---

**Step 1.1**: 招聘方确认匹配，进入群聊

- **操作**: 招聘方在「候选人列表」中点击「接受匹配」，随后系统跳转至群聊页面
- **预期**:
  - 页面右上角显示「已接受匹配，正在等待候选人确认」
  - 候选人一端收到通知气泡：「招聘方已接受，请确认进入薪资协商」
- **验证 AC**:
  - `ISSUE-COM001~c2` — 连接认证（JWT token 验证）
  - `DATE-004~func` — 双方接受引荐后进入四人群聊交友
  - `JOB-004~func` — 双方 Agent 和用户在四人群聊中协商薪资（入口验证）

---

**Step 1.2**: 求职者确认进入群聊

- **操作**: 求职者在通知弹窗中点击「确认加入」，或从匹配列表页点击「进入协商」
- **预期**:
  - 四人群聊界面加载完成，顶部显示房间信息：「薪资协商 · [职位名] · 候选人: 张三 vs HR: 李经理」
  - 消息列表中有系统消息：「已为你们创建薪资协商房间，双方 Agent 已就位」
- **截图验证**: 四人群聊顶部标题 + 系统消息可见
- **验证 AC**:
  - `DATE-004-AC-1` — 四人群聊界面
  - `ISSUE-COM001~c1` — Socket.io 与 Express 集成，房间创建
  - `ISSUE-COM001~c4` — 房间成员查询（验证 4 人均在线）

---

**Step 1.3**: 验证四人群聊成员状态

- **操作**: 观察群聊右上角在线成员列表，悬停查看详情
- **预期**: 显示 4 个成员头像和在线状态：
  - 候选人（人类）— 在线
  - 招聘方（人类）— 在线
  - 候选人 Agent — 在线
  - 招聘方 Agent — 在线
- **截图验证**: 在线成员面板截图，显示 4 人在线
- **验证 AC**:
  - `ISSUE-COM001~c4` — 房间管理（加入/离开房间，房间成员查询）
  - `AS-005-AC-3` — 同时与多个候选保持独立对话（本群 + 其他对话列表共存）

---

**Step 1.4**: 验证 Agent 模式切换入口（招聘方视角）

- **操作**: 招聘方点击聊天输入框左侧「Agent 模式」切换按钮
- **预期**: 按钮变为「亲自聊」/「让 Agent 代聊」两态切换，切换时显示动画过渡
- **截图验证**: 切换前后状态截图
- **验证 AC**:
  - `US-AGENT-007~func` — 用户在四人群聊中成功切换「亲自聊」和「让 Agent 代聊」

---

### Phase 2: Agent 驱动初始薪资提议

**目的**: 验证 AI Agent 基于市场数据和双方 JD 自动生成薪资建议，双方 Agent 在群聊中发表提议

---

**Step 2.1**: Agent 自动生成薪资提议

- **操作**: 等待约 5–10 秒（Agent 生成延迟），观察群聊消息自动出现
- **预期**:
  - 候选 Agent 发送：「根据您的期望和 JD 薪资范围 15K–25K，建议首轮报价 18K，已参考市场同岗中位数。」
  - 招聘方 Agent 发送：「感谢候选人报价，我方首轮 offer: 17K，包含 13 薪。」
- **截图验证**: 两条 Agent 消息截图，带时间戳
- **验证 AC**:
  - `ISSUE-JOB004~c1` — 协商房间（协商话题管理：薪资）
  - `ISSUE-JOB004~c2` — AI 协商（基于市场数据的薪资建议，协商策略生成）
  - `JOB-004-AC-1` — Agent 间进行薪资范围协商

---

**Step 2.2**: 验证薪资调整提议组件

- **操作**: 招聘方在候选 Agent 报价消息下方点击「调整提议」按钮
- **预期**: 弹出薪资调整滑块组件（范围 15K–25K），支持输入具体数字和福利备注
- **截图验证**: 薪资调整组件截图
- **验证 AC**:
  - `ISSUE-JOB004~c6` — UI实现（薪资调整提议组件）

---

**Step 2.3**: 招聘方发出counter-offer

- **操作**: 在薪资调整组件中填入 19K，点击「发送提议」
- **预期**: 消息列表中出现招聘方人类消息：「我方提高至 19K，含 14 薪，欢迎考虑。」发送后招聘方 Agent 自动补充：「已将您的反馈纳入分析，对方提薪至 19K，接近建议区间，建议考虑。」同时候选 Agent 自动回复候选人的参考意见
- **截图验证**: counter-offer 消息 + Agent 分析截图
- **验证 AC**:
  - `ISSUE-JOB004~c2` — AI 协商（条件让步建议，多轮协商支持）
  - `ISSUE-JOB004~c1` — 协商进度追踪（多轮对话历史）

---

### Phase 3: 候选人发起薪资请求

**目的**: 验证候选人一侧的人机交互，候选 Agent 根据候选人意图调整策略

---

**Step 3.1**: 候选 Agent 自动分析当前形势

- **操作**: 等待 Agent 自动响应（约 5 秒内）
- **预期**: 候选 Agent 发送：「当前对方 offer 19K，含 14 薪，折合年薪约 26.6 万。距您的期望 22K 还差 3K。建议：可提出要求增加技能补贴或提前转正。」
- **截图验证**: 候选 Agent 策略分析截图
- **验证 AC**:
  - `ISSUE-JOB004~c2` — AI 协商（双方利益分析与平衡点寻找，协商结果总结）

---

**Step 3.2**: 候选人亲自发出最终请求

- **操作**: 候选人将 Agent 模式切换为「亲自聊」，在输入框输入：「我接受 19K+14薪，但希望增加每月 1500 的技能补贴，能否安排？」点击发送
- **预期**: 消息成功发送，进入人类消息流（无 Agent 标识）
- **截图验证**: 候选人消息截图
- **验证 AC**:
  - `US-AGENT-007~func` — 切换「亲自聊」（候选人侧）
  - `ISSUE-COM001~c3` — 消息实时同步（Socket.io 广播，候选人发出后招聘方立即可见）

---

**Step 3.3**: 招聘方 Agent 快速响应

- **操作**: 等待约 3–5 秒
- **预期**: 招聘方 Agent 发送：「收到，技能补贴 1500/月 可行，请确认是否以此为基础达成初步一致？」
- **截图验证**: Agent 响应截图
- **验证 AC**:
  - `AS-005-AC-2` — 自主发起对话请求（Agent 间主动交互）
  - `ISSUE-JOB004~c2` — AI 协商（协商策略生成）

---

### Phase 4: 协商确认与面试安排

**目的**: 验证双方达成一致后进入面试安排流程，包含日历选择和确认

---

**Step 4.1**: 候选人确认薪资条件

- **操作**: 候选人在招聘方 Agent 消息下方点击「接受条件」，或输入「同意」确认
- **预期**: 弹出协商结果确认框，显示最终条款：薪资 19K + 14薪 + 技能补贴 1500/月。点击「确认」
- **截图验证**: 协商结果确认弹窗截图
- **验证 AC**:
  - `ISSUE-JOB004~c4` — 结果处理（协商结果确认机制，双方同意检测）

---

**Step 4.2**: 系统自动记录协商结果

- **操作**: 确认后等待页面响应
- **预期**:
  - 群聊中出现系统消息：「双方已就薪资达成一致，记录已保存。」
  - 协商历史面板（可在右上角展开）显示：协商时间线、各轮报价、最终条款
- **截图验证**: 协商历史面板截图
- **验证 AC**:
  - `ISSUE-JOB004~c1` — 历史协商记录
  - `ISSUE-JOB004~c4` — 协商记录导出

---

**Step 4.3**: 面试时间协调器出现

- **操作**: 协商结果确认后，等待面试安排 UI 自动加载
- **预期**:
  - 群聊中招聘方 Agent 发送：「恭喜！请选择您偏好的面试时间」
  - 群聊下方出现面试时间选择器（可选择：视频/电话/现场）
  - 日历视图显示双方可用时间（蓝色为共同空闲）
- **截图验证**: 面试时间选择器截图
- **验证 AC**:
  - `ISSUE-JOB004~c3` — 面试安排（面试时间协调，日历集成）
  - `ISSUE-JOB004~c6` — UI实现（面试时间选择器）

---

**Step 4.4**: 候选人选择面试时间

- **操作**: 候选人在日历中选择一个双方空闲的时间段（如周三 14:00），选择「视频面试」，点击「提交」
- **预期**:
  - 消息列表中出现面试提议：「候选人提议：周三 14:00 视频面试，请招聘方确认」
  - 招聘方一侧出现「确认/调整」按钮
- **截图验证**: 面试时间提议消息截图
- **验证 AC**:
  - `ISSUE-JOB004~c3` — 面试安排（面试时间提议/确认）

---

**Step 4.5**: 招聘方确认面试

- **操作**: 招聘方点击「确认」
- **预期**:
  - 群聊中出现系统消息：「面试已确认：周三 14:00，视频面试」
  - 双方日历中均出现面试日程（带提醒）
  - 群聊状态变为「协商完成·面试已安排」
- **截图验证**: 面试确认消息 + 群聊状态截图
- **验证 AC**:
  - `ISSUE-JOB004~c3` — 面试确认与提醒
  - `ISSUE-JOB004~c4` — 人机切换触发（进入真人面试）
  - `JOB-004-AC-2` — 达成初步一致后引荐真人面试

---

### Phase 5: 协商僵局异常处理

**目的**: 验证当薪资差距无法弥合时，系统正确处理协商僵局并给出合理建议

---

**Step 5.1**: 进入新协商会话（重置场景）

- **操作**: 以另一组候选人和招聘方身份重新进入群聊协商流程（可通过测试数据或重新发起匹配）
- **预期**: 重新进入 Phase 1–2 状态（四人群聊 + Agent 初始提议）

---

**Step 5.2**: 触发协商僵局

- **操作**: 候选人要求 30K（超出 JD 最高范围 25K），招聘方 counter offer 15K，双方各持己见超过 3 轮
- **预期**:
  - 系统检测到僵局（约第 4 轮时）：候选 Agent 发送：「检测到双方立场差距较大，建议：候选人考虑调整预期至 22K，或招聘方考虑上调至 20K+绩效奖金」
  - 若继续僵持，系统消息：「当前薪资差距 10K，建议进入下一阶段面试后再评估，或由双方 HR 介入人工协商」
- **截图验证**: 僵局提示截图
- **验证 AC**:
  - `JOB-004~edge` — 异常：协商僵局、薪资超出底线
  - `ISSUE-JOB004~c2` — AI 协商（双方利益分析与平衡点寻找）

---

**Step 5.3**: 验证协商记录导出

- **操作**: 僵局状态下，招聘方点击右上角「协商记录」→「导出」
- **预期**:
  - 可导出 PDF 或 JSON 格式的完整协商记录（包含各轮报价、时间戳、最终僵局原因）
- **验证 AC**:
  - `ISSUE-JOB004~c4` — 协商记录导出
  - `ISSUE-JOB004~c1` — 历史协商记录

---

### Phase 6: 群聊退出与重连

**目的**: 验证用户在任意阶段可随时退出群聊，系统正确处理连接生命周期

---

**Step 6.1**: 候选人在协商中途退出

- **操作**: 候选人在群聊右上角点击「退出协商」，在确认弹窗中点击「确定退出」
- **预期**:
  - 群聊消息列表中出现系统消息：「候选人已退出协商」
  - 候选人端跳转回匹配列表页
  - 招聘方 Agent 自动发送：「候选人已退出，如需重新匹配请返回列表」
- **截图验证**: 退出确认弹窗 + 系统退出消息截图
- **验证 AC**:
  - `US-AGENT-015-AC-1` — 在匹配列表页或四人群聊中可随时退出
  - `US-AGENT-015~func` — 用户在匹配过程中随时退出
  - `ISSUE-COM001~c3` — 断开连接清理
  - `ISSUE-COM001~c4` — 离开房间广播

---

**Step 6.2**: 招聘方重连验证

- **操作**: 招聘方刷新页面（模拟断网重连），观察房间状态恢复
- **预期**:
  - 刷新后自动重新加入同一协商房间
  - 历史消息完整加载
  - 在线成员列表正确显示剩余成员
  - 无需手动重新进入
- **截图验证**: 重连后房间状态截图
- **验证 AC**:
  - `ISSUE-COM001~c3` — 重连机制
  - `ISSUE-COM001~c3` — 心跳检测（重连期间保活）
  - `ISSUE-COM001~c4` — 房间成员查询（重连后一致性）

---

**Step 6.3**: 频繁退出加入异常检测

- **操作**: 候选人在 2 分钟内连续退出 3 次并重新进入
- **预期**:
  - 第 3 次进入后弹出提示：「检测到频繁退出，请确认是否继续？」（防抖限制）
  - 多次退出记录在后台留存（供审核）
- **截图验证**: 频繁退出警告截图
- **验证 AC**:
  - `US-AGENT-015~edge` — 异常：频繁退出加入
