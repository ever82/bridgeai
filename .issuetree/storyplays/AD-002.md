# Story Playbook: 配置商家 Agent 画像

**Story**: `AD-002` | **Narrative**: 商家配置 Agent，设置目标客户画像、优惠活动

## 验收旅程概览

跟随商家「张老板」在 AgentAd 场景中完成商家 Agent 的完整配置：从基础画像录入，到目标客户分层设定，再到优惠活动创建，随后配置四级信息披露策略，激活 Agent 并让其自主匹配潜在客户，最终通过聊天界面审核 Agent 发送的推销消息、处理客户还价。整个旅程以真实商家视角驱动，自然覆盖 L1/L2/L3 信息模型、聊天房间机制、信息披露控制三大系统板块的验收标准。

## 前置条件

- 后端服务、Postgres/PostGIS、Redis 已启动 (`pnpm db:migrate && pnpm db:seed`)
- 已注册测试商家账号「张老板」并登录 mobile 应用
- 张老板已完成 US-AGENT-001 的基础绑定（手机号、昵称、头像）
- AgentAd 场景处于「未配置」状态
- 消费者账号「小李」已注册，AgentAd 场景已配置目标客户为「25-35岁女性」，用于匹配测试
- 第二测试账号「王商家」用于跨商家边界验证
