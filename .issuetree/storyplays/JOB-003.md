# Story Playbook: Agent 简历筛选（带过滤）

**Story**: `JOB-003` | **Narrative**: Agent 间自动匹配职位和简历，双方过滤条件都满足才推荐。

## 验收旅程概览

本剧本模拟真实双端用户的有机旅程：招聘方 "HR-小李" 发布职位并等待匹配结果，求职者 "候选人-小王" 上传简历并查看智能推荐。旅程从账号准备贯穿到匹配通知，覆盖匹配算法、AI 筛选、推荐引擎、搜索接口、通知系统和移动端 UI 的有机验证。

## 前置条件

1. 已安装 BridgeAI mobile app (iOS/Android)
2. 已有 BridgeAI 账号并完成基础认证
3. 数据库已迁移 (`pnpm db:migrate`) 并包含测试用户数据 (`pnpm db:seed`)
4. 服务端运行: `pnpm dev` (apps/server, apps/mobile)
5. 测试账号 (招聘方): `test+recruiter@bridgeai.com` / `password123`
6. 测试账号 (求职者): `test+candidate@bridgeai.com` / `password123`
7. 已创建招聘方 Agent 和求职者 Agent (参见 AS-JOB-002)

### Phase 1: 招聘方发布职位（候选方上传简历）

**目的**: 验证双向过滤条件录入，为后续匹配引擎提供输入数据

  - 技能要求: "React, TypeScript, Node.js"

  - 薪资范围: "25k-40k"
  - 地域: "杭州-西湖区"

### Phase 3: 双向筛选与快速操作

**目的**: 验证招聘方和求职者的交互操作、推荐反馈学习机制

**Step 3.1**: 招聘方快速筛选 - 感兴趣 / 跳过

- 预期:
  - "感兴趣": 候选人进入下一轮深度筛选池，状态变为 "已感兴趣"
  - "跳过": 候选人移出推荐列表（但仍在简历库中）
  - 列表自动刷新，下一个推荐出现
- 验证 AC: `ISSUE-JOB003b~c2` 推荐反馈学习 (喜欢/跳过)
- 验证 AC: `ISSUE-JOB003d~c1` 快速操作 (感兴趣/跳过)
- 截图: `10-quick-actions-recruiter.png`

**Step 3.2**: 求职者快速操作 - 感兴趣 / 跳过

- 操作: 求职者推荐列表 -> 对职位 A 点击 "投递" -> 对职位 B 点击 "暂不感兴趣"
- 预期:
  - "投递": 简历推送给招聘方，状态变为 "已投递"
  - "暂不感兴趣": 职位移出推荐池
  - 推荐列表刷新
- 验证 AC: `AS-JOB-002~func` 招聘方 Agent 筛选简历并与候选人 Agent 初筛

- 操作: 再次刷新推荐列表 -> 确认之前被 "跳过" 或 "投递" 的职位/候选人不重复出现
- 预期:
  - 列表中无重复记录
  - 已被操作过的条目正确过滤
- 验证 AC: `ISSUE-JOB003b~c2` 推荐去重与刷新

### Phase 4: 异常与边界处理

**目的**: 验证单向匹配、过滤条件变更等边界场景的健壮性

**Step 4.1**: 单向匹配场景 - 招聘方条件宽松但求职方严格

- 操作:
  1. 招聘方修改职位薪资上限为 "10k-15k"（低价）
  2. 求职者期望薪资仍为 "20k-35k"（高价）
  3. 观察匹配结果
- 预期:
  - 系统不应推荐该职位给该求职者（薪资不匹配）
  - 匹配度评分中薪资维度显示红色低分或 0%
  - 招聘方查看 "智能推荐" 中该候选人消失
- 验证 AC: `JOB003~edge` 异常：单向匹配、过滤条件变更
- 截图: `13-salary-mismatch-no-match.png`

**Step 4.2**: 过滤条件变更 - 招聘方收紧条件

- 操作:
  1. 招聘方将经验要求从 "3-5年" 改为 "5年以上"
  2. 重新进入匹配流程
- 预期:
  - 之前匹配度较高的候选人（如 1 年经验）评分下降
  - 招聘方推荐列表重新排序
  - 候选人收到 "您的匹配职位有变动" 通知
- 验证 AC: `JOB003~edge` 过滤条件变更
- 截图: `14-filter-changed-resort.png`

**Step 4.3**: 无匹配结果场景

- 操作: 招聘方设置极端过滤条件: "博士 / 10年以上经验 / 期望薪资 100k+" -> 观察匹配结果
- 预期:
  - 推荐列表显示空状态页
  - 文案: "暂无匹配候选人，试试放宽条件？"
  - 提供 "放宽条件" 快捷入口
- 验证 AC: `JOB003~edge` 异常场景的 UI 友好提示
- 截图: `15-no-match-empty-state.png`
