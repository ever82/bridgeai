# Story Playbook: 接收 Agent 私聊建议

**Story**: `US-AGENT-009`
**Narrative**: Agent 给自己发送私聊建议，仅自己可见。

## 验收旅程概览

验证用户在与对方互动时能收到 Agent 的私聊建议，建议仅用户自己可见，不影响对话流程。

## 前置条件

- 服务已启动
- 数据库已迁移
- 用户已创建 Agent 并配置至少一个场景
- 用户已进入某场景的对话中

## 旅程步骤

### Phase 1: Agent 私聊建议触发与显示

**目的**: 验证 Agent 在对话中能主动发送私聊建议

**Step 1.1**: 进入对话页面

- 操作: 用户打开某场景下的进行中对话
- 预期: 对话页面正常加载，底部输入框和 Agent 助手入口可见

**Step 1.2**: 触发 Agent 建议

- 操作: 用户收到对方消息后，等待 Agent 分析并生成建议
- 预期: 页面侧边或底部出现 Agent 私聊建议气泡，标注"Agent 建议"或类似标识

**Step 1.3**: 验证建议仅自己可见

- 操作: 确认建议区域与对话消息区域分离，建议内容不会发送给对方
- 预期: 对方无法看到 Agent 建议内容，仅用户自己可见

**Step 1.4**: 采纳建议

- 操作: 点击建议中的"采纳"按钮
- 预期: 建议内容自动填入输入框，用户可编辑后发送

**Step 1.5**: 忽略建议

- 操作: 关闭或忽略建议气泡
- 预期: 建议消失，不影响正常对话流程

### Phase 2: 异常场景

**目的**: 验证私聊建议的边界情况

**Step 2.1**: Agent 建议为空

- 操作: 在对话内容过于简短时触发建议
- 预期: Agent 显示"暂无建议"或不显示建议区域

**Step 2.2**: 网络中断时建议加载

- 操作: 在弱网环境下等待 Agent 建议
- 预期: 显示加载状态，网络恢复后正常显示建议
