**Story**: `AS-001` | **Narrative**: 主人创建我，我成为该场景专属的 Agent。

| 阶段                | 核心操作                         | 覆盖 Issue                                     |
| ------------------- | -------------------------------- | ---------------------------------------------- |
| Phase 1: 身份初始化 | 用户注册、设置昵称头像、选择场景 | ISSUE-C001                                     |
| Phase 2: Agent 诞生 | 创建 Agent、向导配置、初始画像   | ISSUE-C001                                     |

## 旅程步骤

### Phase 1: 身份初始化

**目的**: 验证用户画像创建和 Agent 创建入口

**Step 1.1**: 用户注册与基础画像设置

- 操作: 打开应用 → 点击「注册」→ 输入手机号/邮箱 → 完成验证码验证
- 预期:
  - 注册成功，显示欢迎页
  - 自动跳转至「完善资料」页面
- 验证 AC:
  - `US-AGENT-001-AC-1` 用户设置昵称和头像
  - `US-AGENT-001-AC-2` 用户选择感兴趣的场景（可多选）

**Step 1.2**: 上传头像与昵称

- 操作: 点击头像区域 → 选择相册图片/拍照 → 裁剪确认 → 输入昵称「小明」
- 预期:
  - 头像显示为上传图片
  - 昵称「小明」保存成功
- 验证 AC:
  - `US-AGENT-001-AC-1` 用户设置昵称和头像
  - `US-AGENT-001~func` 用户首次创建 Agent 并配置基本信息和偏好

**Step 1.3**: 选择感兴趣的场景

- 操作: 在场景选择页面 → 勾选「AgentDate」(交友) → 点击「下一步」
- 预期:
  - 页面显示 4 个场景: VisionShare, AgentDate, AgentJob, AgentAd
  - 至少选中 1 个场景后可继续
- 验证 AC:
  - `US-AGENT-001-AC-2` 用户选择感兴趣的场景（可多选）

**Step 1.4**: 配置沟通风格偏好

- 操作: 滑动选择沟通风格 → 选择「详细」风格 → 点击「完成」
- 预期:
  - 显示 4 种风格: 直接/委婉/详细/简洁
  - 选择后高亮显示
- 验证 AC:
  - `US-AGENT-001-AC-3` 用户配置 Agent 沟通风格

### Phase 3: 性格塑造

**目的**: 验证 Agent 学习场景规则、生成性格行为准则、配置过滤系统

**Step 3.1**: Agent 学习场景过滤规则

- 操作: 进入「小明助手」详情页 → 点击「开始学习」→ 等待 AI 处理
- 预期:

- 截图验证: Agent 学习完成页面

**Step 3.2**: 配置地域过滤

- 操作: Agent 详情页 → 点击「过滤设置」→ 进入地域过滤面板
- 预期:

- 截图验证: 地域过滤设置面板
- 验证 AC:

  - `ISSUE-C003~c2` 位置选择 UI - 三级联动选择器
  - `ISSUE-C003~c3` 距离筛选 - 距离范围筛选

**Step 3.3**: 设置具体地域过滤

- 操作: 地域过滤面板 → 选择「北京市-朝阳区」→ 距离「10km」→ 保存
- 预期:

- 验证 AC:

- 截图验证: 属性过滤设置面板
- 验证 AC:
  - `ISSUE-C004~c1` 查询 DSL - 条件表达式定义、操作符
  - `ISSUE-C004~c3` 过滤 UI - 筛选器面板、条件添加/删除

**Step 3.5**: 添加属性过滤条件

- 操作: 属性过滤 → 添加条件: 年龄范围 25-35岁 → 性别「不限」→ 保存
- 预期:
  - 条件卡片显示「年龄 25-35」
  - 列表更新，显示符合条件的候选
- 验证 AC:
  - `ISSUE-C004~c1` 查询 DSL - 逻辑组合 AND, OR
  - `ISSUE-C004~c4` 过滤接口 - 按属性筛选 Agent

**Step 3.6**: 配置信用过滤

- 操作: 过滤设置 → 进入「信用过滤」面板 → 设置最低门槛
- 预期:
  - 显示信用分滑块选择器

- 截图验证: 信用过滤设置面板

### Phase 5: 异常考验

**目的**: 验证边界情况和异常处理

**Step 5.1**: 网络中断处理

- 操作: 开启飞行模式 → 尝试创建 Agent → 查看错误处理
- 预期:
  - 显示网络错误提示
  - 提供「重试」按钮
  - 不出现白屏或崩溃
- 验证 AC:
  - `US-AGENT-001~edge` 异常场景：网络中断

**Step 5.2**: 无效输入处理

- 操作: 正常网络下 → 创建 Agent → 名称输入为空 → 点击创建
- 预期:
  - 显示字段错误提示「名称不能为空」
  - 阻止提交
- 验证 AC:
  - `US-AGENT-001~edge` 异常场景：无效输入

**Step 5.3**: 主人画像缺失处理

- 操作: 使用无昵称/无头像的账号 → 创建 Agent → 查看 Agent 表现
- 预期:
  - 显示警告「画像不完整」
  - 允许继续但 Agent 能力受限
- 验证 AC:
  - `AS-001~edge` 异常：主人画像缺失

**Step 5.4**: 注销 Agent 场景

- 操作: Agent 详情页 → 点击「注销」→ 确认注销
- 预期:
  - 显示注销确认对话框
  - 注销后 Agent 状态变为「archived」
  - 历史数据保留
- 截图验证: 注销确认对话框
- 验证 AC:
  - `US-AGENT-017-AC-1` 注销特定场景 Agent
  - `US-AGENT-017-AC-2` 保留历史交易记录

**Step 5.5**: 注销后重建

- 操作: 注销完成 → 返回 Agent 列表 → 重新创建同名 Agent
- 预期:
  - 可以重新创建
  - 新 Agent 无历史数据
- 验证 AC:
  - `US-AGENT-017~edge` 异常：注销后重新创建

**Step 5.6**: 场景规则加载失败

- 操作: 模拟后端规则服务不可用 → Agent 尝试学习 → 查看降级处理
- 预期:
  - 显示加载失败提示
  - Agent 使用默认规则
  - 提供「重试」选项
- 验证 AC:
  - `AS-001~edge` 异常：场景规则加载失败

**Step 5.7**: 过滤后无候选

- 操作: 设置极窄过滤条件 (如: 信用分 900+ 且 10km 内) → 查看候选列表
- 预期:
  - 显示「未找到候选」空状态
  - 提供「放宽条件」建议
- 验证 AC:
  - `AS-DATE-001~edge` 异常：过滤后无候选

**Step 5.8**: 信用不足处理

- 操作: 查看某候选卡片 → 该候选信用分为 400 → 查看提示
- 预期:
  - 显示「信用不足」标签
  - 提供提升信用建议
- 验证 AC:
  - `ISSUE-C005~c4` 不足处理 - 提升信用建议
