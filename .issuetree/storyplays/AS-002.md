# Story Playbook: 创建主人的需求

**Story**: `AS-002` | **Narrative**: 主人用自然语言告诉我他想要什么，我提炼成结构化需求。

## 验收旅程概览

本剧本以"小明（求职者 + 交友者 + 视觉爱好者）"和"小红（HR + 商家）"两位主人为视角，
覆盖从首次配置 Agent、用自然语言表达需求/供给、Agent 自动提炼三层信息模型、
激活实时匹配、Agent 间协商对话、向主人请求引荐、群聊代聊、举报/退出的完整生命周期。

旅程顺序遵循**真实使用路径**：注册→配置（L1/L2/L3）→自然语言对话→AC 自动提取→确认→
存储/激活→匹配查询→Agent 间协商→建议→披露策略→引荐→群聊→退出/举报，并在每个阶段
混入对应场景（VisionShare/AgentDate/AgentJob/AgentAd）的差异化覆盖与异常路径。

- 总 Phase: 9
- 总 Step: 32
- 唯一 AC 数: 146
- 覆盖场景: 通用 + VS + Date + Job + Ad

## 环境启动

```bash
# 1) 安装依赖
cd /Users/z/projects/bridgeai
pnpm install

# 2) 启动数据库（需要本地 Postgres + PostGIS 已就绪）
pnpm db:migrate
pnpm db:seed

# 3) 启动后端 + 移动端（Expo Web）
pnpm dev    # turbo 并行启动 apps/server 与 apps/mobile

# 4) 单独运行端到端验收（可选）
pnpm test:e2e:web      # web 端到端
pnpm test:e2e          # mobile 端到端

# 5) 构建生产
pnpm build
```
