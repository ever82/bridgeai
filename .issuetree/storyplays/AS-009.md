# Story Playbook: 我请求主人确认引荐

**Story**: `AS-009`
**Narrative**: 匹配度达标后请求主人确认引荐。

## 验收旅程概览

模拟一位用户 "小明" 通过 Agent 交友场景的完整旅程：从创建 Agent、Agent 自主发起对话、实时私聊建议、匹配度达标后收到引荐请求、双方确认引荐、进入四人群聊，到最终学习偏好并影响后续推荐。旅程中自然覆盖 Agent 对话引擎、信用分、协商机制、记忆系统等全部 AC。

## 前置条件

- 服务已启动（`pnpm dev`）
- 数据库已迁移（`pnpm db:migrate` + `pnpm db:seed`）
- 已注册两个测试用户：小明（User A）和小红（User B）
- 两人均已完成个人资料填写，信用分初始化
- 两人均已创建约会场景的 Agent，并配置了人格和偏好
- AI 对话服务可用（LLM API 正常）
- WebSocket 服务已启动，支持实时推送

## 旅程步骤

### Phase 1: Agent 出场 -- 创建与配置

**目的**: 验证 Agent 创建、人格定义、场景配置的完整性（ISSUE-AI004 核心能力）

**Step 1.1**: 用户小明打开 Agent 列表，创建新 Agent

- 操作: 小明打开 App -> 进入 Agent 管理页面 -> 点击 "创建 Agent" -> 填写 L1 基础信息（昵称"桥小助"、场景选择"交友约会"）-> 填写 L2 个人资料 -> 点击 "下一步"
- 预期: 创建 Agent 页面各步骤可正常切换，L1ProfileForm、L2ProfileForm 表单验证通过，Agent 创建成功后出现在列表中
- 验证 AC: `ISSUE-AI004~c2` 人格系统 -- Agent人格定义、语气风格适配、行为特征模拟、人格一致性维护

**Step 1.2**: 配置 Agent 的 AI 人格与对话风格

- 操作: 小明在 L3TextEditor 中编写 Agent 自我介绍与对话风格描述，选择沟通风格为 "friendly"，在 SceneConfigForm 中选择交友场景配置
- 预期: AIConfigSection 显示已配置的人格参数，Agent 预览卡片（AgentPreview）展示 Agent 人格摘要
- 验证 AC: `ISSUE-AI004~c2` 场景角色扮演、人格一致性维护；`ISSUE-AI004~c1` 引擎上下文管理基础

**Step 1.3**: 查看信用分初始状态

- 操作: 小明打开个人设置 -> 查看信用分面板（CreditScore 组件）
- 预期: 显示当前信用分分数、等级（如 "良好"）、等级描述、各维度评分因子
- 验证 AC: `ISSUE-CR001~c3` 信用等级划分机制；`ISSUE-CR001~c4` GET /api/v1/credit/score 接口返回正确数据；`ISSUE-CR001~c1` 信用分数据库设计（CreditScore 表、CreditHistory 表、CreditFactor 表）

### Phase 2: Agent 自主出击 -- 发起多路对话

**目的**: 验证 Agent 自主决定发起对话、同时与多候选保持独立对话、协商机制

**Step 2.1**: 小明的 Agent 自主发现并联系候选 Agent

- 操作: 小明在首页看到 Agent 状态卡片显示 "正在为您寻找匹配..."，等待 Agent 通过 matchQueryService 筛选出符合条件的候选（使用 optimizedMatchingService 进行优化匹配）
- 预期: Agent 状态变为 "已发现 3 位候选，正在交流中"，列表中显示小红等候选人的 Agent 信息
- 验证 AC: `AS-005~func` Agent 自主决定与符合条件的候选 Agent 发起对话；`AS-005-AC-2` 自主发起对话请求；`AS-VS-001-AC-3` 筛选信用高、距离近的候选（结合信用分过滤）

**Step 2.2**: Agent 同时与多个候选保持独立对话

- 操作: 进入 Agent 对话监控页面，看到 Agent 与 3 位候选同时进行独立对话（agentInitiatedChat 服务），每个 ChatSession 有独立的消息流和状态
- 预期: 每个对话窗口显示独立的对话内容，状态栏显示各对话进度（如 "交流中 / 第3轮"），对话间互不干扰
- 验证 AC: `AS-005-AC-3` 同时与多个候选保持独立对话

**Step 2.3**: 验证 Agent 对话引擎核心能力

- 操作: 查看其中一段 Agent-Agent 对话日志，确认 dialogMessage 中包含 senderType="agent"、intent 字段、confidence 分数、contextSummary
- 预期: 对话消息结构完整，包含意图识别结果、上下文摘要，对话轮次按 maxAutoTurns 控制，轮次间有 turnDelayMs 延迟
- 验证 AC: `ISSUE-AI004~c1` 引擎实现 -- 上下文管理、意图识别、回复生成、对话状态跟踪、多轮对话支持；`ISSUE-AI004~c6` API实现 -- 对话生成API、WebSocket实时对话

**Step 2.4**: Agent 被其他 Agent 联系后评估

- 操作: 小红的 Agent 同时也在寻找匹配，小红的 Agent 主动联系了小明的 Agent。小明的 Agent 收到对话请求后，通过 agentNegotiationService 评估是否值得交流
- 预期: 小明的 Agent 自动回复接受对话，状态显示 "对方 Agent 主动联系，已评估并接受"
- 验证 AC: `AS-006~func` Agent 被其他 Agent 联系后评估是否值得交流；`AS-006-AC-2` 评估是否值得交流；`ISSUE-AI004~c3` 协商机制 -- 双向Agent对话、匹配意愿确认

### Phase 3: 智能助攻 -- 实时私聊建议与意图分析

**目的**: 验证 Agent 在对话过程中实时分析、向主人汇报、提供策略建议

**Step 3.1**: 小明查看 Agent 实时私聊建议

- 预期: 私聊建议区域仅小明可见（其他对话参与者不可见），包含话题推荐、风险提示、对方意图分析
- 验证 AC: `US-AGENT-009~func` Agent 实时向用户发送私聊建议，仅自己可见；`US-AGENT-009-AC-1` 私聊建议显示在侧边栏或独立区域；`US-AGENT-009-AC-2` 建议包括话题推荐、风险提示、对方意图分析

**Step 3.2**: 验证实时意图分析与策略建议

- 操作: 查看私聊建议中的具体内容，确认包含：对方意图标签（如 "友好 / 探索中"）、风险提示（如 "对方回应频率较低"）、话题推荐（如 "聊聊共同兴趣"）
- 预期: 建议内容结构化、有针对性，随对话进展实时更新
- 验证 AC: `AS-008-AC-1` 实时分析对方意图和态度；`AS-008-AC-2` 发送策略建议、风险提示、话题推荐；`AS-008~func` Agent 实时分析对话，向主人发送私下建议；`AS-006-AC-3` 向主人汇报对话进展

**Step 3.3**: 切换亲自聊 / Agent 代聊

- 操作: 小明点击 "亲自聊" 按钮，Agent 暂停自动对话，小明以自己身份发送一条消息；然后点击 "让 Agent 聊" 按钮，Agent 恢复自动对话
- 预期: 切换过程平滑，对话上下文完整移交，消息发送者标识清晰变化（AgentIdentity 组件显示身份），对方能看到发送者类型变化
- 验证 AC: `US-AGENT-007~func` 用户在四人群聊中成功切换 "亲自聊" 和 "让 Agent 代聊"；`US-AGENT-007-AC-2` 点击 "让 Agent 聊" 按钮后 Agent 以自己名义发言；`ISSUE-AI004~c5` 切换机制 -- 人工介入检测、对话上下文移交、恢复AI代理

### Phase 4: 高潮 -- 匹配度达标，请求引荐确认

**目的**: 验证匹配度评估、引荐请求、双向确认机制、信用分展示

**Step 4.1**: Agent 评估匹配度达到引荐阈值

- 操作: Agent 与小红 Agent 完成若干轮对话后，matchScore 从初始值上升至 85 分（超过 highMatchThreshold = 80），Agent 自动判断匹配度达标，暂停对话
- 预期: Agent 状态显示 "匹配度达标，正在请求主人确认引荐"，matchData 中 matchScore >= 引荐阈值
- 验证 AC: `AS-009-AC-1` 评估匹配度是否达到引荐阈值；`AS-009~func` 匹配度达标后 Agent 请求主人确认引荐

**Step 4.2**: 小明收到引荐通知，查看对方信息和信用分

- 操作: 小明收到推送通知 "您的 Agent 推荐了一位匹配对象"，点击进入 ReferralScreen 页面
- 预期: 引荐页面（ReferralScreen）完整显示：
  - 匹配度分数徽章（如 "匹配度 85%"）
  - Agent 对话摘要
  - 匹配亮点（compatibilityFactors 标签列表）
  - 推荐话题
  - 对方信用分（CreditBadge 组件）
  - 决策倒计时
- 验证 AC: `US-AGENT-006-AC-1` 收到引荐通知，显示匹配理由；`US-AGENT-006-AC-2` 显示对方关键信息和信用分；`AS-009-AC-2` 向主人展示对方信息和信用分；`AS-PROTO-001-AC-1` 每条消息包含发送者类型和信用分

**Step 4.3**: 小明同意引荐

- 操作: 小明点击 "同意" 按钮，确认弹窗出现，点击 "确定"
- 预期: API 调用 POST /dating/referrals/:id/decision 成功，页面显示 "您已同意引荐。等待对方决策..."，倒计时继续运行
- 验证 AC: `US-AGENT-006~func` Agent 请求引荐真人时，用户能接受或拒绝

**Step 4.4**: 小红也同意引荐，双向匹配成功

- 操作: 切换到小红账号，小红也收到引荐通知，在 ReferralScreen 上看到对方信息和信用分，点击 "同意"
- 预期: 双方同意后，MutualConsent result 变为 MUTUAL_ACCEPT，ReferralRecord status 变为 SUCCESS，系统自动创建四人群聊房间
- 验证 AC: `DATE-004~func` 双方接受引荐后进入四人群聊交友；`DATE-004-AC-2` Agent 提供交友话题建议
