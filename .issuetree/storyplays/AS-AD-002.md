# Story Playbook: AgentAd 优惠发现与使用

**Story**: `AS-AD-002`
**Narrative**: 用户通过 AgentAd 场景的 Agent 发现和使用优惠。

## 验收旅程概览

跟随消费者「小美」完成从发现优惠、评估匹配、发起匹配、协商优惠、到完成成交的完整旅程。商家「张老板」的 Agent 主动推销优惠，小美通过自己的 Agent 接收、协商并最终使用优惠。整个旅程以真实的双向匹配 + Agent 代聊协商为主线，自然覆盖供给映射、智能评估、匹配度计算、协商成交等 AC。

## 前置条件

- 后端服务、Postgres/PostGIS、Redis 已启动 (`pnpm db:migrate && pnpm db:seed`)
- 已注册测试商家账号「张老板」，其 AgentAd 场景已激活，配置了「春季数码节」优惠活动（满1000减150，预算上限5000元/天，最低折扣底线9折），目标客户：25-35岁、深圳、中高端消费
- 已注册测试消费者账号「小美」，AgentAd 场景已配置品类偏好「数码3C」、预算100-200元/天、品牌偏好「苹果/华为/小米」
- 小美和张老板已完成 US-AGENT-001 的基础绑定（手机号、昵称、头像）
- 双方 Agent 均处于「活跃」状态，小美 Agent 读取了主人画像并生成了场景性格（「关注性价比·数码爱好者」）
- 第二消费者账号「小强」已注册，AgentAd 配置为「运动户外·低价敏感」，用于无匹配场景验证

### 设计原则

1. **一个完整旅程**: 从发现优惠 → 评估匹配 → 发起匹配 → 协商成交 → 使用核销，完整覆盖双向 Agent 交互链路。
2. **自然覆盖 AC**: 每个 Phase 自然串联多个 AC，例如 Phase 3 (Agent推销) 同时验证 `AS-AD-002~func` + `US-AGENT-008-AC-2` + `ISSUE-C007-AC-1`。
3. **真实操作**: 每步都是可执行的 App 操作（点击按钮、查看面板、发送消息），非假设性描述。
4. **可截图验证**: 每个关键节点提供截图点，确保验收可追溯（25张截图点）。
5. **包含异常处理**: Phase 6 专项覆盖 5 类异常场景（无匹配、还价超底线、用户拒绝、品类不匹配、预算耗尽）。
