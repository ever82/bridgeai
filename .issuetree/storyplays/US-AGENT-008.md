# Story Playbook: 在四人群聊中感知对方身份

**Story**: `US-AGENT-008`
**Narrative**: 用户在四人群聊中感知对方是人类还是 Agent。

## 验收旅程概览

用户 Alice 与 Bob 在 AgentDate 交友场景完成匹配后，双方接受引荐进入四人交友群聊（Alice、Agent-Alice、Bob、Agent-Bob）。旅程聚焦于 Alice 视角的完整感知体验：从进入群聊的第一眼观察消息历史中的身份标识，到两人对话中不断感知对方身份，再到 Bob 切换 Agent 模式后 Alice 实时收到通知并看到标识更新。涵盖正常路径与异常路径，覆盖 UI 组件底层基础设施的验证。

## 前置条件

- 服务已启动（`pnpm dev`）
- 数据库已迁移（`pnpm db:migrate`）
- 已创建两个测试用户 Alice 和 Bob（各配一个 Agent）
- Alice 与 Bob 的 Agent 已完成初筛匹配并请求引荐
- 两个用户的移动端模拟器/真机均已登录并进入同一个四人交友群聊
- 群聊初始状态：Alice 和 Bob 均以 Agent 模式发言

### Phase 5: 异常感知 — 标识稳定性

**目的**: 验证在网络异常、快速操作等场景下，身份感知标识保持稳定

**Step 5.1**: 快速连续切换的身份感知稳定性

- 操作: Bob 在 2 秒内快速连续切换身份 3 次（用户→Agent→用户→Agent）
- 预期:
  - 系统使用防抖（debounce）机制，仅保留最后一次稳定状态
  - Alice 收到的切换通知最多显示一次（200ms 防抖窗口内只保留最终状态）
  - 消息标识稳定在最终状态（Agent 模式）
  - 不出现混乱的标识组合（如"Bob"+"紫色气泡"）
- 验证 AC:
  - `US-AGENT-008~edge` — 异常：身份标识错乱（确认不发生）
  - `US-AGENT-007~edge` — 异常：切换中断、并发冲突
  - `ISSUE-COM002~c4` — 状态管理：已读回执、在线/离线状态（状态稳定性）
- 可截图: 防抖后稳定状态截图

## 备注

- **感知核心**: 本旅程以 Alice 视角为感知主体，观察 Bob（用户/Bob 的 Agent）的消息身份标识变化。感知维度包括：消息发送者名称（带/不带"Agent"后缀）、头像边框颜色（Agent 紫色/用户蓝色）、消息气泡颜色、系统通知横幅。
- **senderSnapshot 机制**: 每条消息在创建时通过 `createSenderSnapshot()` 捕获发送者身份快照（`senderSnapshot`），存储在消息数据中。`resolveMessageSender()` 函数优先使用快照而非实时数据，确保历史消息的感知标识不随后续身份切换而改变。
- **颜色语义**: 代码中定义的 Agent 标识颜色为紫色 `#7C3AED`（`IDENTITY_VISUALS.AGENT.background`），用户标识颜色为蓝色 `#2563EB`。气泡颜色使用 `SENDER_TYPE_COLORS`（Agent 紫色/用户蓝色）。
- **防抖保护**: `SenderChangeIndicator` 组件内部实现了 150ms 防抖（`timerRef`），快速连续切换时只显示最终稳定状态，防止 UI 抖动。
- **Socket 房间广播**: 身份切换通过 Socket.io 事件（如 `group:state_sync`）实时广播到群聊所有成员，实现 Alice 和 Bob 的双向感知。
- **AC 覆盖**: 旅程中完整验证了 US-AGENT-008 的 3 个核心 AC 和 2 个 func/edge AC，并通过 Phase 7 横向验证了 Job 和 Ad 场景的身份感知一致性。
- **异常感知**: Phase 5 通过防抖测试、网络重连、认证失败等场景，验证身份感知在异常情况下的稳定性和可恢复性。
