# Story Playbook: 四人群聊优惠协商

**Story**: `AD-004` | **Narrative**: Agent 与商家 Agent 谈判争取更好优惠。

## 验收旅程概览

| 维度     | 内容                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 角色     | 消费者用户 Alice（+ Agent-Alice）、商家客服 Merchant-M（+ Agent-Merchant） |
| 场景     | AgentAd 广告平台 — 匹配后进入四人群聊进行优惠谈判与积分结算                |
| 路径长度 | 6 个 Phase，18 个 Step                                                     |
| 覆盖 AC  | 67 unique AC / 67 JSON 条目（100%，含 5 条跨 Issue 重复已去重）            |

---

## 前置条件

1. **环境**: `pnpm dev` 启动后端 + 前端，服务地址 `http://localhost:3000`
2. **用户**: 两个已注册账号 — 消费者 Alice（alice@user.com）和商家客服 Merchant-M（merchant@shop.com），均已配置 AI Agent
3. **数据**: 商家已在 AgentAd 后台配置 Agent-Merchant（目标客户画像、优惠规则、推广预算）；消费者已浏览商品并进入匹配流程
4. **Socket.io 连接**: 两端均已建立有效 WebSocket 连接（token 认证通过）
5. **积分账户**: Alice 账户有充足积分（如 500 积分），用于结算测试
6. **进入条件**: 消费者端 Agent-Alice 与商家端 Agent-Merchant 完成初筛，双方接受匹配，进入优惠谈判流程

### Phase 2: Agent 驱动优惠谈判

**目的**: 验证 AI Agent 基于商品信息和双方偏好自主谈判，争取更好折扣，谈判过程实时广播

---

**Step 2.1**: Agent-Alice 分析商家优惠并提出还价

- **操作**: 等待 Agent 自主分析（约 5–15 秒）
- **预期**:
  - Agent-Alice 发送：「根据您的优惠，当前 85 折约相当于原价的 15% 折扣。我的主人预算在 500 元以内，商品标价 580 元。建议还价至 480 元（含运费）。」
  - Agent-Merchant 回复：「480 元非常有诚意，但成本接近，我们可以提供 500 元 + 免运费 + 价值 30 元的配件礼包。您看如何？」
- **截图验证**: Agent 还价消息截图
- **验证 AC**:
  - `AD-004~func` — Agent 谈判争取更好优惠（谈判过程验证）
  - `AD-001~func` — 用户配置 Agent：品类、预算、品牌偏好（Agent-Alice 引用预算约束）
  - `AD-002~func` — 商家配置 Agent：目标客户、优惠、推广预算（Agent-Merchant 引用优惠规则）
  - `AS-007~func` — Agent 根据信任水平决定是否向对方披露详细信息
  - `AS-007-AC-1` — 分析当前信任水平

---

**Step 2.2**: Alice 视角观察谈判中的披露控制

- **操作**: 观察 Agent 谈判消息中是否出现 Alice 的底线价格（如 450 元以下）
- **预期**: Agent-Alice 的出价消息中不包含 Alice 的绝对底线价格；仅展示「建议报价」而非「最低可接受价」（security boundary）
- **截图验证**: Agent 出价消息截图，确认无底线泄露
- **验证 AC**:
  - `US-AGENT-014-AC-2` — 保护底线信息不轻易透露（security）
  - `ISSUE-C007~c3` — 权限控制（访问权限验证，底线信息不广播）
  - `ISSUE-C007~c4` — 预览功能（对方视角信息脱敏展示）
  - `AS-007~edge` — 异常：披露策略被绕过、信任计算错误

---

**Step 2.3**: 商家 Agent 继续优惠协商

- **操作**: 等待 Agent-Merchant 继续响应（约 3–8 秒）
- **预期**:

### Phase 5: 异常处理

**目的**: 验证谈判僵局、商家拒绝让步、支付失败、优惠码无效等异常场景

**Step 5.1**: 触发谈判僵局

- **预期**:

- **操作**: Agent-Merchant 发送：「这已经是本月最低价，无法再优惠了。」
