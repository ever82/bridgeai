# Story Playbook: 在四人群聊中切换身份

**Story**: `US-AGENT-007`
**Narrative**: 用户在四人群聊中随时切换"亲自聊"或"让 Agent 代聊"。

## 验收旅程概览

用户 Alice（通过其 Agent-Alice）与 Bob（通过其 Agent-Bob）完成 Agent 间初筛匹配后，双方接受引荐进入四人交友群聊。Alice 在群聊中多次切换身份模式，并在关键节点采纳 Agent 建议，最终礼貌结束对话。旅程从需求创建（Phase 0）开始，覆盖身份切换、实时状态同步、私聊建议、引荐确认、举报与退出等 AC。

## 前置条件

- 服务已启动（`pnpm dev`）
- 数据库已迁移（`pnpm db:migrate`）
- 已创建两个测试用户 Alice 和 Bob（各配一个 Agent）
- Alice 与 Bob 的 Agent 已完成初筛匹配并请求引荐
- 两个用户的移动端模拟器/真机均已登录

## 旅程步骤

### Phase 0: 需求创建 — 与 Agent 对话建立需求/供给

**目的**: 验证用户与 Agent 对话后成功创建需求或供给，并自动保存到对应场景

**Step 0.1**: 用户与 Agent 聊天建立需求

- 操作: Alice 在 AgentDate 场景中与 Agent-Alice 进行对话，说明自己的交友偏好（年龄范围、兴趣标签、期望的伴侣特质）
- 预期: Agent-Alice 理解并归纳 Alice 的需求；对话结束后弹出确认框"已为您创建交友需求，是否保存？"
- 验证 AC:
  - `US-AGENT-002~func` — 用户与 Agent 对话后成功创建需求和供给
  - `US-AGENT-002-AC-5` — 用户确认后自动保存到对应场景
- 可截图: 需求确认对话框截图

**Step 0.2**: 异常场景 — 空输入或无意义输入

- 操作: Alice 输入空白消息或"asdfgh"等无意义文字
- 预期: Agent 回复"请提供更多信息"；若连续多次无效输入，Agent 礼貌提示并结束本次对话；系统不崩溃
- 验证 AC:
  - `US-AGENT-002~edge` — 异常场景：空输入、无意义输入、超长对话
- 可截图: Agent 无效输入回复截图

**备注**: Phase 0 可在任何场景（AgentDate/AgentJob/AgentAd）执行，其目的是在进入四人群聊前建立用户需求/供给档案。Phase 0 执行后，Agent 才能据此与对方 Agent 进行有意义的初筛对话（Phase 1 的前置条件）。

### Phase 3: 核心体验 — 身份模式切换

**目的**: 验证用户在四人群聊中成功切换"亲自聊"和"让 Agent 代聊"，切换状态实时显示

- 预期:
  - 身份指示器从"🤖 智能体"（紫色）变为"👤 用户"（蓝色），带有切换动画（淡入淡出）

  - 后续 Alice 发送的消息显示为"Alice"（无 Agent 后缀）

- 验证 AC:
  - `US-AGENT-007-AC-1` — 点击"亲自聊"按钮后以用户身份发言
  - `US-AGENT-007-AC-3` — 切换状态实时显示在聊天室

## 备注

- **身份切换防抖**: 代码实现中使用了 200ms debounce 防止快速连续切换导致的 UI 抖动（参见 `ChatScreen.tsx` 中的 `identitySwitchTimerRef`）
