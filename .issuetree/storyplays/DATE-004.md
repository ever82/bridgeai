# Story Playbook: 四人群聊交友沟通

**Story**: `DATE-004`
**Narrative**: 双方接受引荐后，进入四人群聊进行交友沟通。

## 验收旅程概览

本剧本以一对真实用户（Alice 与 Bob）和他们各自的 Agent（Agent-Alice、Agent-Bob）作为主线人物，从需求建立 → Agent 间初筛对话 → 引荐确认 → 四人群聊交友 → 身份切换 → Agent 实时建议 → 披露控制 → 退出/举报 的完整有机旅程，验证交友（AgentDate）主场景，并横向延展到求职（AgentJob 薪资协商）和广告（AgentAd 优惠谈判）两个同构场景。旅程同时下钻到底层基础设施（Socket.io 通信、聊天房间、消息 UI 组件）确保交付质量。

## 前置条件

- 服务已启动（`pnpm dev`），数据库已迁移（`pnpm db:migrate`）
- 已创建 4 个测试账号：Alice（求偶用户）、Bob（求偶用户）、Carol（HR/求职用户，复用横向场景）、Merchant-M（商家用户）
- 各账号已创建对应 Agent 并完成 onboarding
- 两台移动端设备/模拟器同时在线（Alice 设备 A、Bob 设备 B），可观察对方端实时变化
- 后端 Socket.io 命名空间 `/chat`、`/presence` 可达
- 测试用 Redis（用于 Socket adapter）正常工作

## 环境启动

```bash
# 1. 安装依赖
pnpm install

# 2. 数据库迁移
pnpm db:migrate

# 3. 启动开发服务（后端 + 前端）
pnpm dev

# 4. 在两台模拟器（Alice/Bob）启动 mobile app
cd apps/mobile && pnpm ios   # 或 pnpm android

# 5. （可选）E2E 测试
pnpm test:e2e         # Maestro
pnpm test:e2e:web     # 浏览器流程

# 6. 手动验证关键路径：
#    Phase 0  AgentDate 中与 Agent 聊天 → 创建需求并保存
#    Phase 1  后台触发 Agent 间初筛对话
#    Phase 2  接受引荐 → 进入四人群聊
#    Phase 3  观察消息身份、状态、列表性能
#    Phase 4  切换"亲自聊" ↔ "Agent 代聊"
#    Phase 5  接收 Agent 私聊建议、采纳、否决披露
#    Phase 6  AgentJob/AgentAd 横向场景
#    Phase 7  举报、礼貌结束、退出
#    Phase 8  历史回看
#    Phase 9  断网、并发、协议元数据校验
```

## 备注

- **多设备协同**: Phase 4.3、5.1、7.3 等步骤需双设备同时观察才能验证"对方实时收到"。
- **底线保护**: AS-JOB-001-AC-3（security）的关键校验点是 Agent-Alice 内部薪资底线绝不能在协议层泄露给对方 Agent，可通过抓包对照 Agent 出站消息字段来验证。
- **私聊建议隔离**: Phase 5.3 的"建议泄露"防护需双端抓包对比，确认 `agent:private_advice` 通道仅推送给主人 socket。
- **异常覆盖**: 19 个 `~edge` 类型 AC 在 Phase 0.2、1.1、1.2、1.3、2.3、4.5、5.3、6.1、6.2、7.3、8.2、9.1 中分别覆盖。
- **基础设施层**: ISSUE-COM001/COM002/UI004/UI004a 的子条目（c1-c6）通过有机用户操作自然触发，无需独立测试用例。
- **覆盖度**: 99/99 = 100%。
