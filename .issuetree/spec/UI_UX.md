# BridgeAI - UI/UX 设计文档

## 1. 设计原则

### 1.1 核心设计理念

- **透明优先**: 用户始终知道在跟谁对话（Agent 还是真人）
- **控制感**: 用户随时可切换人机，掌控对话节奏
- **信任建立**: 信用分、评分、身份标识清晰可见
- **场景化**: 不同场景有不同的信息架构和视觉风格

### 1.2 交互原则

| 原则 | 说明 | 体现 |
|------|------|------|
| 即时反馈 | 切换身份时实时同步 | 对方界面立即显示切换提示 |
| 清晰标识 | Agent 和真人视觉区分明确 | 机器人头像 vs 真人头像 |
| 渐进披露 | 敏感信息按需展示 | 私聊建议仅自己可见 |
| 一对多并行 | 同时与多个候选交流 | 标签页切换不同群聊 |

## 2. 全局导航系统

### 2.1 底部导航栏（移动端）

```
┌─────────────────────────────────────────────┐
│                                             │
│           [当前页面内容区域]                   │
│                                             │
│                                             │
├─────────────────────────────────────────────┤
│  🏠      🎯      💬      👤      ⚙️        │
│ 首页    场景    消息    我的    设置         │
│            [未读角标]  [红点]                │
└─────────────────────────────────────────────┘
```

**导航项说明**:
- **首页**: 个人 Agent 状态、信用分、积分、推荐匹配
- **场景**: 四大场景入口（VisionShare / AgentDate / AgentJob / AgentAd）
- **消息**: 所有群聊会话列表、系统通知
- **我的**: 个人资料、Agent 配置、需求/供给管理
- **设置**: 过滤规则、隐私设置、帮助中心

### 2.2 顶部状态栏

```
┌─────────────────────────────────────────────┐
│ BridgeAI  ≡                    🔍    🔔  💎 │
│                                             │
│  [页面标题]                          [操作]  │
└─────────────────────────────────────────────┘
```

**状态栏元素**:
- 左侧: Logo + 汉堡菜单（展开侧边导航）
- 右侧: 搜索、通知（红点提示）、积分余额
