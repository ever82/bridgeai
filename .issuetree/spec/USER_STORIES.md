# BridgeAI 用户故事

## 通用 Agent 功能

### US-AGENT-001: 创建个人 Agent
**角色**: 用户
**描述**: 用户首次使用 BridgeAI，创建自己的 AI Agent，配置基本信息和偏好。
**验收标准**:
- [ ] 用户设置昵称和头像
- [ ] 用户选择感兴趣的场景（可多选）
- [ ] 用户配置 Agent 沟通风格（直接/委婉/详细/简洁）
- [ ] Agent 生成初始画像

### US-AGENT-002: 通过与 Agent 对话创建需求和供给
**角色**: 用户
**描述**: 用户通过与自己的 Agent 自然语言对话，让 Agent 帮助创建或更新需求和供给信息，无需手动填写表单。
**验收标准**:
- [ ] **启动对话创建**:
  - [ ] 用户可通过文字或语音告诉 Agent 自己的需求或能提供的帮助
  - [ ] Agent 主动追问关键信息（如场景类型、具体条件、底线等）
  - [ ] 支持多轮对话逐步完善信息
- [ ] **需求信息创建**（各场景）:
  - [ ] VisionShare: "我想看某医院现在人多不多" → Agent 创建视觉需求
  - [ ] AgentDate: "我想找一个喜欢爬山、年龄 25-30 岁的对象" → Agent 创建交友需求
  - [ ] AgentJob: "我想找一份远程的前端工作，薪资期望 20k+" → Agent 创建求职需求
  - [ ] AgentAd: "我想买一台 5000 左右的笔记本电脑" → Agent 创建购物需求
- [ ] **供给信息创建**（各场景）:
  - [ ] VisionShare: "我现在在某商场，可以帮别人拍照" → Agent 创建视觉供给
  - [ ] AgentDate: "我可以提供我的基本信息和照片" → Agent 创建交友供给
  - [ ] AgentJob: "我们公司在招 Python 工程师，薪资 15-25k" → Agent 创建招聘供给
  - [ ] AgentAd: "我有一批闲置数码产品想出售" → Agent 创建广告供给
- [ ] **智能信息提取**:
  - [ ] Agent 从对话中自动提取关键字段（地点、时间、预算、要求等）
  - [ ] Agent 将信息分类为公开/详细/私密三级
  - [ ] 敏感信息（如底价、底线）自动标记为私密，仅自家 Agent 可见
- [ ] **确认与修改**:
  - [ ] Agent 总结提取的信息向用户确认
  - [ ] 用户可以修改、补充或删除字段
  - [ ] 确认后自动保存到对应场景的需求/供给配置

### US-AGENT-005: 查看 Agent 对话记录
**角色**: 用户
**描述**: 用户查看自己的 Agent 与其他 Agent 的对话历史，了解匹配过程。
**验收标准**:
- [ ] 按场景筛选对话记录
- [ ] 显示对话双方 Agent
- [ ] 显示匹配度评分
- [ ] 显示对方信用分和评分
- [ ] 可查看完整对话内容

### US-AGENT-006: 接受/拒绝引荐
**角色**: 用户
**描述**: 当 Agent 匹配成功并请求引荐真人时，用户决定是否接受。
**验收标准**:
- [ ] 收到引荐通知，显示匹配理由
- [ ] 显示对方关键信息（脱敏）
- [ ] 显示对方信用分和评分记录
- [ ] 可选择接受或拒绝
- [ ] 接受后双方可开始真人沟通

### US-AGENT-007: 在四人群聊中切换身份
**角色**: 用户
**描述**: 用户在四人群聊（甲用户、甲 Agent、乙用户、乙 Agent）中，随时可以切换"亲自聊"或"让 Agent 代聊"。
**验收标准**:
- [ ] 聊天界面显示四个参与者：甲 Agent、乙 Agent、对方用户、自己
- [ ] 点击"亲自聊"按钮后，自己以用户身份发言
- [ ] 点击"让 Agent 聊"按钮后，Agent 以自己名义发言
- [ ] 切换状态实时显示在聊天室
- [ ] 对方能看到"对方已切换到 Agent/用户模式"

### US-AGENT-008: 在四人群聊中感知对方身份
**角色**: 用户
**描述**: 用户在四人群聊中，始终能清楚看到每条消息是谁发的（用户 or Agent）。
**验收标准**:
- [ ] 每条消息显示头像和名称（Agent 机器人头像、用户真人头像）
- [ ] Agent 消息标识为"XX 的 Agent"或"Agent 昵称"
- [ ] 用户消息标识为用户昵称
- [ ] 对方切换时实时更新消息发送者标识
- [ ] 聊天室顶部显示当前各方状态（谁在亲自聊、谁是 Agent 在聊）

### US-AGENT-009: 接收 Agent 私聊建议
**角色**: 用户
**描述**: 在四人群聊过程中，自己的 Agent 给自己发送私聊建议，仅自己可见。
**验收标准**:
- [ ] 私聊建议显示在侧边栏或独立区域
- [ ] Agent 实时分析对话并提供建议
- [ ] 建议包括：话题推荐、风险提示、对方意图分析
- [ ] 可一键采用建议（Agent 自动发送该内容到群聊）
- [ ] 可忽略建议

### US-AGENT-010: 真人交流后评分
**角色**: 用户
**描述**: 双方进入真人交流并完成互动后，可以对对方进行评分，评价多个维度。
**验收标准**:
- [ ] 交流结束后弹出评分界面
- [ ] 可对诚信度评分（1-5星）
- [ ] 可对礼貌度评分（1-5星）
- [ ] 可对响应度评分（1-5星）
- [ ] 可对整体满意度评分（1-5星）
- [ ] 可写文字评价（可选）
- [ ] 评分影响对方信用分

### US-AGENT-011: 配置需求信息（分公开和详细）
**角色**: 用户
**描述**: 用户配置自己在不同场景下的需求信息，分为公开信息（对方 Agent 可浏览）和详细信息（仅自家 Agent 掌握）。
**验收标准**:
- [ ] **VisionShare**: 
  - [ ] 公开：想看某地什么情况
  - [ ] 详细：具体地点、期望角度、预算上限
- [ ] **AgentDate**: 
  - [ ] 公开：想找什么类型、兴趣标签
  - [ ] 详细：年龄范围、收入期望、房车要求
- [ ] **AgentJob** (求职者): 
  - [ ] 公开：技能方向、工作年限
  - [ ] 详细：薪资底线、公司偏好
- [ ] **AgentAd**: 
  - [ ] 公开：想买什么品类、预算区间
  - [ ] 详细：品牌偏好、急需程度、底价
- [ ] 为每个场景设置披露策略（何时披露详细信息）

**字段分级说明**:
| 层级 | 可见范围 | 示例字段 | 用途 |
|------|---------|---------|------|
| queryable | 系统索引 | 坐标、时间、类别 | 用于快速筛选匹配 |
| readable | 对方 Agent | 描述、偏好、要求 | 用于 Agent 间评估匹配度 |
| private | 仅自家 Agent | 预算底线、敏感信息 | 用于谈判策略和保护底线 |

### US-AGENT-012: 配置供给信息（分公开和详细）
**角色**: 用户
**描述**: 用户配置自己在不同场景下的供给信息，分为公开信息（对方 Agent 可浏览）和详细信息（仅自家 Agent 掌握）。
**验收标准**:
- [ ] **VisionShare**: 
  - [ ] 公开：在附近、能拍照
  - [ ] 详细：精确位置、设备类型、可拍摄时间
- [ ] **AgentDate**: 
  - [ ] 公开：基本类型、兴趣标签
  - [ ] 详细：真实年龄、收入、房产、约会偏好
- [ ] **AgentJob** (招聘方): 
  - [ ] 公开：招聘什么岗位、技能要求
  - [ ] 详细：薪资上限、团队氛围
- [ ] **AgentAd** (商家): 
  - [ ] 公开：有什么商品、大概价格
  - [ ] 详细：成本价、最大优惠空间、库存
- [ ] 为每个场景设置展示策略（何时展示详细能力）

**字段分级说明**:
| 层级 | 可见范围 | 示例字段 | 用途 |
|------|---------|---------|------|
| queryable | 系统索引 | 位置、可服务时间、类别 | 用于快速筛选匹配 |
| readable | 对方 Agent | 能力列表、特点、条件 | 用于 Agent 间评估匹配度 |
| private | 仅自家 Agent | 成本结构、最低价格、库存 | 用于谈判策略和保护底线 |

### US-AGENT-013: 浏览对方公开需求和供给
**角色**: 用户
**描述**: 用户通过自家 Agent 浏览对方的公开需求信息和供给信息，判断是否值得进一步交流。
**验收标准**:
- [ ] Agent 展示对方公开的需求概要
- [ ] Agent 展示对方公开的供给概要
- [ ] 显示对方的信用评分作为参考
- [ ] 显示匹配度评估（需求 vs 供给的契合度）
- [ ] 用户决定是否发起交流或继续浏览其他

### US-AGENT-014: Agent 控制信息披露
**角色**: 用户
**描述**: 在四人群聊中，自家 Agent 根据对话情况决定是否向对方披露详细信息。
**验收标准**:
- [ ] Agent 分析对话进展和对方可信度
- [ ] 支持两种披露模式：
  - [ ] **每次确认模式**: 每次披露前向用户发送提醒，等待用户确认
  - [ ] **自动模式**: 根据预设策略自动披露，用户保留实时否决权
- [ ] 保护底线信息不轻易透露
- [ ] 用户可实时否决即将发生的披露

### US-AGENT-015: 退出匹配流程
**角色**: 用户
**描述**: 用户在匹配过程中的任何阶段，可以主动退出当前匹配流程。
**验收标准**:
- [ ] 在匹配列表页可随时退出当前匹配
- [ ] 在四人群聊中可随时退出对话
- [ ] 退出时 Agent 礼貌结束对话并告知对方
- [ ] 退出后该匹配进入历史记录
- [ ] 可选择屏蔽对方，不再接收其匹配

### US-AGENT-016: 举报不当行为
**角色**: 用户
**描述**: 用户遇到不当行为时，可向平台举报，平台进行处理。
**验收标准**:
- [ ] 可在对话界面举报对方 Agent 或用户
- [ ] 可选择举报类型（骚扰、欺诈、虚假信息等）
- [ ] 可上传截图等证据
- [ ] 举报后平台人工审核
- [ ] 审核结果通知举报人
- [ ] 被举报方信用分根据情节扣减

### US-AGENT-017: 注销 Agent
**角色**: 用户
**描述**: 用户可注销某个场景的 Agent，清除相关数据。
**验收标准**:
- [ ] 可选择注销特定场景的 Agent
- [ ] 注销前确认未完成的对话和交易
- [ ] 注销后清除该场景的需求/供给配置
- [ ] 保留历史交易记录（用于信用追溯）
- [ ] 可重新创建该场景的 Agent

## VisionShare 场景

### VS-001: 配置视觉需求
**角色**: 用户
**描述**: 用户发布视觉需求，设置响应者的过滤规则。
**验收标准**:
- [ ] 用户输入自然语言描述（地点+意图）
- [ ] 系统解析出地点和意图
- [ ] 设置 VisionShare 专属过滤：照片时效性要求
- [ ] 系统只向符合过滤条件的附近用户推送任务
- [ ] 系统显示发布的任务和过滤条件

### VS-002: 附近任务接单（带过滤）
**角色**: 用户
**描述**: 用户打开接单界面，系统根据用户设置的过滤规则（距离、信用分等）显示符合条件的附近任务。
**验收标准**:
- [ ] 只显示符合地域过滤条件的任务
- [ ] 只显示符合信用过滤条件的任务
- [ ] 显示任务发布者的信用分
- [ ] 点击接单后进入拍照界面
- [ ] 拍照并上传后获得积分奖励

### VS-003: AI 智能检索本地相册
**角色**: 用户
**描述**: 系统智能扫描用户本地相册，发现与当前需求地点匹配且拍摄时间较近的照片时，推荐给用户确认上传。
**验收标准**:
- [ ] 系统筛选 GPS 地点在需求地点附近的照片
- [ ] 系统筛选拍摄时间较近的照片
- [ ] AI 分析照片内容是否包含需求场景特征
- [ ] 符合条件的照片以推荐形式展示给用户
- [ ] 用户确认后上传并获得积分奖励

> **说明**: 此为实验性功能，首期可用系统推荐替代自动扫描。

### VS-004: AI 隐私脱敏处理
**角色**: 用户
**描述**: 照片上传前，AI 自动进行隐私脱敏处理，包括人脸模糊/打码和去除敏感元数据。
**验收标准**:
- [ ] AI 自动检测并模糊处理照片中的人脸
- [ ] 去除照片 EXIF 元数据中的敏感信息（精确 GPS 坐标、设备 ID 等）
- [ ] 脱敏后的照片预览给用户确认
- [ ] 用户可选择是否发布脱敏后的照片

### VS-005: 查看照片（显示对方信用）
**角色**: 用户
**描述**: 用户支付积分查看照片，界面显示照片拍摄者的信用分和评分。
**验收标准**:
- [ ] 照片列表页显示价格、缩略图和拍摄者信用分
- [ ] 支付积分后显示完整照片
- [ ] 显示拍摄者详细评分记录
- [ ] 可以对照片和拍摄者评分

### VS-006: 历史照片查询
**角色**: 用户
**描述**: 用户想知道"去年五一这家医院人多不多"，输入后看到当时的历史照片。
**验收标准**:
- [ ] 支持按地点和时间搜索
- [ ] 返回匹配的历史照片
- [ ] 显示拍摄时间和拍摄者信用分

> **说明**: 此功能需要积累足够的历史数据，标记为二期功能。

## AgentDate 场景

### DATE-001: 配置交友 Agent 画像
**角色**: 用户
**描述**: 用户配置 AgentDate 场景下的交友偏好。
**验收标准**:
- [ ] 设置个人基本信息（年龄、性别、婚恋状态等）
- [ ] 设置期望对方的条件（年龄范围、性别、婚恋状态）
- [ ] 设置隐私级别（什么信息可被 Agent 披露）

### DATE-002: Agent 主动匹配推荐（带过滤）
**角色**: 用户
**描述**: 用户的 Agent 根据画像和过滤规则自动寻找符合条件的匹配对象。
**验收标准**:
- [ ] Agent 只扫描符合地域过滤条件的用户
- [ ] Agent 只扫描符合属性过滤条件的用户
- [ ] Agent 只扫描符合信用过滤条件的用户
- [ ] 计算匹配度评分
- [ ] 向用户推送推荐通知，显示匹配理由和对方信用分

### DATE-003: Agent 间对话匹配
**角色**: 用户
**描述**: 双方 Agent 先进行对话，了解彼此，匹配度高后请求双方主人同意引荐。
**验收标准**:
- [ ] Agent 间可进行多轮对话
- [ ] 对话内容记录并可查看
- [ ] Agent 评估匹配度
- [ ] 显示对方实时状态（Agent/用户在线）
- [ ] 匹配度达标后请求引荐

### DATE-004: 在四人群聊中进行交友沟通
**角色**: 用户
**描述**: 双方接受引荐后，进入四人群聊界面，可进行交友沟通。
**验收标准**:
- [ ] 四人群聊界面，显示四方头像和名称（参考 US-AGENT-007/008 通用功能）
- [ ] Agent 提供交友话题建议（如共同兴趣）
- [ ] 约会意向达成后可约定时间和地点

### DATE-005: 约会后评分
**角色**: 用户
**描述**: 约会完成后，双方对彼此进行评分，影响信用分。
**验收标准**:
- [ ] 约会结束提醒评分（参考 US-AGENT-010 通用评分功能）
- [ ] 评分维度增加：约会体验
- [ ] 评分匿名显示给对方
- [ ] 评分实时影响信用分

## AgentJob 场景

### JOB-001: 配置求职者 Agent 画像
**角色**: 求职者
**描述**: 求职者配置 Agent，包括技能、经验、期望薪资、工作偏好。
**验收标准**:
- [ ] 设置技能标签和熟练度
- [ ] 设置工作经历
- [ ] 设置期望薪资范围
- [ ] 设置工作偏好（远程/onsite、公司规模等）

### JOB-002: 配置招聘方 Agent 画像
**角色**: 招聘方 (HR/用人部门)
**描述**: 招聘方配置 Agent，发布职位需求。
**验收标准**:
- [ ] 设置职位信息和要求
- [ ] 设置薪资预算范围
- [ ] 设置硬性条件（必须满足的技能/经验）
- [ ] 设置软性偏好（加分项）

### JOB-003: Agent 简历筛选（带过滤）
**角色**: 用户
**描述**: 招聘方 Agent 与求职者 Agent 进行初步匹配，双方过滤条件都满足才推荐。
**验收标准**:
- [ ] Agent 间自动匹配职位和简历
- [ ] 检查双方过滤条件是否满足
- [ ] 评估匹配度（技能、薪资、经验）
- [ ] 显示对方信用分
- [ ] 向双方推荐匹配结果

### JOB-004: 在四人群聊中进行薪资协商
**角色**: 用户
**描述**: 双方 Agent 和用户在四人群聊中就薪资、福利、工作制度等进行协商。
**验收标准**:
- [ ] 四人群聊界面，显示四方：求职者、求职 Agent、HR、HR Agent（参考 US-AGENT-007/008 通用功能）
- [ ] Agent 间进行薪资范围协商
- [ ] 记录双方底线和期望
- [ ] 达成初步一致后引荐真人面试

### JOB-005: 面试后评分
**角色**: 用户
**描述**: 面试完成后，双方对彼此进行评分。
**验收标准**:
- [ ] 面试结束提醒评分（参考 US-AGENT-010 通用评分功能）
- [ ] 求职者评 HR：诚信度、专业度、沟通效率
- [ ] HR 评求职者：诚信度、专业度、匹配度
- [ ] 评分影响双方信用分
- [ ] 高信用求职者优先推荐

## AgentAd 场景

### AD-001: 配置用户广告偏好
**角色**: 用户
**描述**: 用户配置 Agent，告诉 Agent 自己对什么类型的广告感兴趣。
**验收标准**:
- [ ] 设置感兴趣的商品/服务类别
- [ ] 设置预算范围
