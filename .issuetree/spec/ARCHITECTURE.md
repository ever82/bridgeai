### 消息类型与可见性

```typescript
// 群聊消息 - 群内所有人可见
interface GroupMessage {
  id: string;
  conversationId: string;
  sender: {
    userId: string;
    agentId: string;
    type: 'agent' | 'human';      // 是 Agent 发的还是用户发的
    name: string;                  // 显示名称
    avatar: string;
  };
  content: string;
  timestamp: Date;
  metadata: {
    creditScore?: number;
    isHandoff?: boolean;
  };
}

// 私聊建议 - 仅自己可见
interface WhisperMessage {
  id: string;
  toUserId: string;              // 只有这个用户能看到
  fromAgent: string;
  content: string;
  type: 'suggestion' | 'warning' | 'info';
  context: string;               // 基于什么上下文给出的建议
  timestamp: Date;
  actions?: {                    // 快捷操作
    label: string;
    action: string;
  }[];
}
```

### 切换流程

```
用户 A 想从"Agent 代聊"切换到"亲自聊":

1. 用户 A 点击 [👤 亲自聊] 按钮
        │
        ▼
2. 系统发送切换消息到群聊:
   "甲用户已亲自上线"
        │
        ▼
3. 用户 A 的输入框变为"亲自聊天中"
   后续消息以"👤 甲用户"身份发送
        │
        ▼
4. 乙用户和乙 Agent 看到:
   - 新消息来自"👤 甲用户"
   - 系统提示"对方用户已亲自上线"
        │
        ▼
5. 甲 Agent 进入旁听模式:
   - 继续分析对话
   - 给甲用户发送私聊建议
   - 但不主动发言
```

// Agent 过滤规则
interface AgentFilterRules {

    incomeRange?: { min: number; max: number };

  };

    minTransactionCount: number;
