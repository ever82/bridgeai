🚨 阻塞性问题（环境层）

Blocker A: 后端 API 路由全部超时（含已注册账号登录）
POST /api/v1/auth/login、POST /api/v1/auth/register、GET /api/v1/agents/types、GET /api/v1/agents 均连接成功但服务端不响应（status=000，超时）。
仅 GET /health 正常返回 200。
服务器日志：
POST /api/v1/auth/login - - ms - -
说明：请求进入 Express 但路由处理器或中间件链阻塞，从未发送响应。
影响: Playbook Phase 1（登录）即被阻断，整个 21+ 步 UI 旅程无法启动。
Blocker B: 移动端 Expo Web 打包失败
浏览器控制台输出：
[error] Refused to execute script from '/expo/AppEntry.bundle?...' because its MIME type ('application/json') is not executable
[error] Failed to load resource: the server responded with a status of 500
Bundle 错误：Unable to resolve module @babel/runtime/helpers/interopRequireDefault from apps/mobile/src/screens/agent/components/SceneConfigForm.tsx
该模块在根 node_modules 存在但 apps/mobile/node_modules 缺失，Metro 无法解析。
影响: localhost:8081 渲染为完全空白页面，UI 演示无法进行。
Blocker C: Chrome MCP 实例冲突 / 截图为白屏
chrome-devtools-mcp 启动时报 "browser already running"，Demo Agent 杀掉残留进程后重新打开，但因 Blocker B 导致页面渲染为纯白。
现有 1 张截图（01-login.png）确为白屏，无 UI 元素。
