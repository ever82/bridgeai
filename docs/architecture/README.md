## 核心模块

### 1. Agent 系统

Agent 是 VisionShare 的核心组件，负责:
- 用户画像构建
- 需求理解与提取
- 供需匹配算法
- 智能推荐

### 2. 匹配引擎

基于多维度的匹配算法:
- 地理位置 (PostGIS 空间查询)
- 技能/需求标签
- 时间可用性
- 信誉评分
- AI 语义理解

### 3. 通信系统

- 实时消息 (WebSocket)
- 推送通知 (Expo Notifications)
- 邮件通知 (可选)

## 技术决策

### 为什么使用 Monorepo?

- 统一的依赖管理
- 共享类型和工具
- 原子性变更
- 简化协作

### 为什么使用 PostgreSQL + PostGIS?

- 强大的地理空间查询能力
- ACID 事务保证
- 成熟的生态系统
- 与 Prisma ORM 良好集成

### 为什么使用 Redis?

- 高性能缓存
- 支持多种数据结构
- 内置 Pub/Sub (Socket.io)
- 队列支持 (BullMQ)

## 安全考虑

- JWT 认证与刷新令牌
- API 速率限制
- 输入验证 (Zod)
- SQL 注入防护 (Prisma)
- XSS 防护
- CORS 配置
- 敏感数据加密

## 扩展性设计

- 水平扩展: 无状态服务设计
- 数据库: 读写分离、分区
- 缓存: 多级缓存策略
- 消息队列: 异步处理
