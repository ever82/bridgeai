# AgentDate Demo Script

**场景**: AgentDate - AI 交友匹配，让单身用户通过 AI Agent 自动匹配志同道合的朋友。

**演示时长**: 4-6 分钟
**演示账号**: `demo-date@bridgeai.com` / `Demo1234!` (Bob)
**演示环境**: 需预先 Seed 数据，见 `docs/DEMO_DATA.md`

---

## 场景概览

- 用户 Bob 配置交友 Agent 画像（喜欢户外活动、爬山）
- AI Agent 自动扫描符合条件的用户，计算匹配度
- 双方 Agent 先进行对话，互相了解
- 匹配成功后进入四人群聊，用户决定是否引荐真人

---

## 操作步骤与讲解词

### Step 1: 配置交友 Agent 画像 (0:00 - 1:30)

**操作**:

1. 登录 `demo-date@bridgeai.com`
2. 进入 AgentDate 场景
3. 点击"配置 Agent"
4. 设置基本信息：喜欢户外活动、周末经常爬山、科幻电影爱好者
5. 设置期望条件：希望对方也喜欢户外活动
6. 设置隐私级别（详细资料仅匹配成功后公开）

**讲解词**:

> AgentDate 的核心理念是：用户不需要自己去找，AI Agent 会帮你找。
> Bob 只需要告诉他的 Agent 他的兴趣爱好，Agent 就会自动工作。
> 用户还可以设置隐私级别，控制哪些信息何时公开。

---

### Step 2: AI 自动匹配推荐 (1:30 - 2:30)

**操作**:

1. 等待几秒，观察"正在匹配"动画
2. 系统推送推荐列表（显示匹配度 85%）
3. 推荐对象：Carol（喜欢户外、爬山、骑行）

**讲解词**:

> Bob 的 Agent 在后台自动扫描所有用户的交友画像。
> 根据兴趣匹配度计算推荐分数，这里 Carol 的匹配度达到了 85%。
> 整个过程无需用户操作，AI 持续工作，有新匹配随时推送。

---

### Step 3: Agent 间对话 (2:30 - 4:00)

**操作**:

1. 点击"开始对话"
2. 进入 Agent 间对话界面
3. 观察双方 Agent 自动对话：
   - Bob 的 Agent: "你好，我是 Bob 的 Agent。Bob 周末喜欢去香山爬山，你有兴趣一起吗？"
   - Carol 的 Agent: "你好！Bob 的 Agent。我也非常喜欢户外运动，上周刚去了百望山。"
4. 双方 Agent 继续交流，分享更多兴趣

**讲解词**:

> 这是 AgentDate 的关键创新：在真人见面前，先让 AI Agent 互相了解。
> 双方 Agent 会代表用户进行礼貌而有深度的对话。
> 如果对话顺利，匹配度会继续上升；否则 Agent 会礼貌结束，避免尴尬。
> 用户可以在侧边栏看到 Agent 的对话建议。

---

### Step 4: 请求引荐与四人群聊 (4:00 - 5:30)

**操作**:

1. 当匹配度达到阈值（如 80%），系统提示"可以引荐了"
2. 双方确认后，进入四人群聊
3. 四人群聊界面显示：Bob、Carol、以及双方的 Agent
4. 每条消息标识发送者身份（人类或 Agent）
5. Bob 点击"亲自聊"切换为真人发言

**讲解词**:

> 当 Agent 间对话达到一定深度，系统会建议双方是否引荐真人。
> 双方都同意后，进入独特的四人群聊：两个真人 + 两个 AI Agent。
> 用户可以随时切换"亲自聊"或"让 Agent 聊"。
> 界面清楚标识每条消息是谁发的，确保透明度。

---

### Step 5: 约会后评分 (5:30 - 6:00)

**操作**（模拟约会完成后的评分流程）:

1. 点击"结束交流"
2. 弹出评分界面
3. 对 Carol 进行多维度评分（诚信度、礼貌度、响应度、满意度）
4. 提交评分

**讲解词**:

> 交流结束后，双方可以给对方评分。
> 评分是多维度的：诚信度、礼貌度、响应度、满意度。
> 评分直接影响用户的信用分，高信用用户更容易获得优质匹配推荐。

---

## 转场设计

| 转场点   | 描述                                     |
| -------- | ---------------------------------------- |
| 开场     | 展示 BridgeAI 首页，聚焦 AgentDate 卡片  |
| Step 1-2 | 画像配置完成后，淡入"正在匹配"动画       |
| Step 2-3 | 推荐卡片飞入，点击后切换到对话界面       |
| Step 3-4 | Agent 对话滚动，匹配度达到阈值后触发转场 |
| Step 4-5 | 用户点击"结束交流"，淡入评分界面         |
| 结尾     | 展示信用分变化，强调 AI 匹配的价值       |

---

## 注意事项

- 演示前运行 `npm run db:seed:demo` 初始化数据
- Agent 间对话是自动进行的，演示时可加速播放或跳到关键节点
- 如需演示四人群聊身份切换，需要同时在线两个账号（可使用双浏览器或模拟）
- 评分流程可在演示结尾简要带过，不展开
