## 2. 场景卡片规范

每个场景卡片包含：
- **场景图标**: 大图标 + 场景色背景
- **场景名称**: 大号字体 + 场景色
- **场景描述**: 两句话说明核心价值
- **操作按钮**: 2-4个主要操作入口

**操作按钮分组**:
- 需求方操作（左侧）
- 供给方操作（右侧）
- 用颜色或间距区分

---

## 3. 场景色规范

```
VisionShare  #3B82F6  (蓝)  视觉/照片
AgentDate    #EC4899  (粉)  交友/情感
AgentJob     #14B8A6  (青)  职业/商务
AgentAd      #F97316  (橙)  购物/优惠
```

每个场景卡片顶部 4px 色条标识场景类型。

---

## 4. 角色切换说明

用户可在同一身份下使用多个角色：
- **VisionShare**: 需求方（发布需求）/ 供给方（接单拍照）
- **AgentDate**: 交友者（寻找匹配）
- **AgentJob**: 求职者 / 招聘方
- **AgentAd**: 消费者 / 商家

场景卡片同时提供两种角色的入口，用户无需切换身份。
