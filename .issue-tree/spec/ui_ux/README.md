## 🎯 设计原则

### 核心设计理念

- **透明优先**: 用户始终知道在跟谁对话（Agent 还是真人）
- **控制感**: 用户随时可切换人机，掌控对话节奏
- **信任建立**: 信用分、评分、身份标识清晰可见
- **场景化**: 不同场景有不同的信息架构和视觉风格

### 交互原则

| 原则 | 说明 | 体现 |
|------|------|------|
| 即时反馈 | 切换身份时实时同步 | 对方界面立即显示切换提示 |
| 清晰标识 | Agent 和真人视觉区分明确 | 机器人头像 vs 真人头像 + 文字标签 |
| 渐进披露 | 敏感信息按需展示 | 私聊建议仅自己可见（悬浮气泡形式） |
| 防误操作 | 关键操作需确认 | 人机切换二次确认、长按触发 |

## ⚠️ 错误处理

### 完整错误处理规范 (12_errors.md)

覆盖场景：
- 网络错误（断开、弱网、超时）
- 表单验证错误
- 权限错误（相机、位置、存储）
- 操作失败（发送失败、加载失败）
- 系统错误（维护、500错误）
- 空状态设计
- 操作确认（二次确认、危险操作）

错误码规范：
- ERR_NETWORK / ERR_TIMEOUT
- ERR_SERVER_500 / ERR_SERVER_503
- ERR_AUTH_401 / ERR_AUTH_403
- ERR_VALIDATION / ERR_CONFLICT

## 🎨 设计系统 (14_design_system.md)

### 颜色规范
- 主色调（靛蓝、紫、绿、橙、红、蓝）
- 场景色（蓝、粉、青、琥珀）
- 中性色（深色/浅色两套）
- **深色模式完整配色**

### 字体规范
- 字体栈（中英文）
- 字号层级（H1-H3, Body, Caption）
- **行高 1.3-1.6**
- **字重最低 400**

### 图标系统
- 线性图标，2px 描边
- 核心图标语义规范
- Emoji仅作示意，实际使用SVG

### 间距规范
- 基础单位 4px
- 安全区适配
- 组件间距标准

### 响应式断点
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

### 无障碍设计
- 颜色对比度 ≥ 4.5:1
- 触摸目标 ≥ 44px
- 屏幕阅读器支持
- 键盘导航
