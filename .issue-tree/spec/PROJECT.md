# BridgeAI - AI Agent 通信平台

## 项目简介

BridgeAI 是一个让 AI Agent 作为用户代表先行沟通的平台。AI 作为人与人之间的桥梁，先进行筛选、匹配、初步沟通，再将合适的连接引荐给真人。

**核心创新**: 强大的过滤机制确保 Agent 只与符合条件的对象交流，无缝人机切换让双方始终知道在跟谁对话，完善的信用系统让每次真人交流都积累信任。

## 核心理念

- **Agent 先行**: AI 代理先沟通，真人后介入
- **智能匹配**: 基于真实需求和画像精准匹配
- **隐私保护**: 敏感信息由 Agent 控制，用户掌控披露节奏
- **场景通用**: 同一套 Agent 通信协议，支持多种垂直场景
- **过滤精准**: 地域、属性、信用多重过滤
- **人机无缝**: Agent 与用户随时切换，对方实时感知

## 四大核心场景（需求-供给模型）

每个场景都遵循"需求方-供给方"双向信息模型：
- **需求方**: 发布需求信息（想要什么）
- **供给方**: 展示供给信息（能提供什么）
- **API 浏览**: 对方 Agent 可浏览公开的需求/供给信息
- **详情披露**: 深入信息仅自家 Agent 掌握，根据对话情况逐步披露

### 1. VisionShare (视野共享) - 一对多实时匹配

**需求方**: "我想看三里屯现在堵车情况"
- **公开**: 想看某地、大致时间
- **详细**: 精确地点、拍摄角度、预算上限

**供给方**: "我能提供附近照片"
- **公开**: 在附近、有拍摄能力  
- **详细**: 精确位置、设备类型、可拍摄时间

**一对多实时匹配过程**:
1. 用户发布需求 → 系统**立即**查询附近**所有**能拍照的用户（一对多）
2. Agent 筛选出 Top 10 最合适的供给方
3. 需求方可以同时与**多个**供给方 Agent 进入四人群聊
4. 需求方可以看到所有候选人的位置、信用分、历史照片质量
5. 选择最优的响应 → 对方拍照上传 → 完成交易

**示例**: 
- 小明: "三里屯现在堵车吗？"
- 系统:**立即**找到 30 个附近的拍照者（一对多）
- Agent: 筛选出信用分高、距离近的 5 个候选人
- 小明同时与 5 个候选人 Agent 聊天确认谁最方便
- 最终选择最近的小红拍照，支付 10 积分

### 2. AgentDate (智能交友) - 一对多筛选

**需求方**: "我想找喜欢户外的伴侣"
- **公开**: 想找朋友、兴趣标签
- **详细**: 年龄范围、收入期望、房车要求

**供给方**: "我喜欢户外，周末常爬山"
- **公开**: 基本类型、兴趣标签
- **详细**: 真实年龄、收入、房产、约会偏好

**一对多实时匹配过程**:
1. 用户配置交友需求 → 系统**立即**查询**所有**匹配对象（一对多）
2. Agent 根据年龄、兴趣、信用等排序，精选 Top 20
3. 用户可以同时与**多个**匹配对象的四人群聊中了解对方
4. 每个聊天都是独立的：用户+Agent ↔ 对象+Agent
5. 筛选出最合适的 → 双方同意 → 交换联系方式

**示例**:
- 小明 Agent: "主人想找喜欢户外的女生，25-30岁"
- 系统:**立即**找到 50 个匹配对象（一对多）
- Agent: 精选出 10 个高匹配度的
- 小明同时与 3 个女生 Agent 聊天
- 发现小红最合拍 → 双方同意引荐 → 交换微信

### 3. AgentJob (智能招聘) - 双向一对多

**需求方（求职者）**: "想找 Python 后端工作"
- **公开**: 技能方向、工作年限
- **详细**: 薪资底线、公司规模偏好、可到岗时间

**供给方（招聘方）**: "招 Python 后端，25-35k"
- **公开**: 岗位类型、技能要求
- **详细**: 具体薪资上限、团队氛围、项目详情

**一对多实时匹配过程**:
1. 求职者发布简历 → 系统**立即**匹配**所有**合适职位（一对多）
2. 同时，招聘方发布职位 → 系统**立即**匹配**所有**合适候选人（一对多）
3. 双方 Agent 可以同时与**多个**对方聊天
4. 求职者可以同时面试多个公司
5. 招聘方可以同时筛选多个候选人

**示例**:
- 小明（求职）: "3年Python经验，期望25-30k"
- 系统:**立即**找到 20 个匹配职位（一对多）
- 小明 Agent 同时与 5 家公司 HR Agent 聊
- 同时，小红（HR）: "招Python后端，预算25-35k"
- 系统:**立即**找到 30 个候选人（一对多）
- 小红 Agent 同时与 10 个候选人聊
- 双方发现匹配 → 安排面试

### 4. AgentAd (智能广告) - 多对多比价

**需求方（消费者）**: "想买跑鞋，预算 500-800"
- **公开**: 想买跑鞋、预算区间
- **详细**: 品牌偏好、急需程度、最低可接受价

**供给方（商家）**: "新款跑鞋 699，限时优惠"
- **公开**: 有跑鞋、价格 699
- **详细**: 成本价、最大优惠空间、库存情况

**一对多实时匹配过程**:
1. 用户发布购买需求 → 系统**立即**匹配**所有**相关商家（一对多）
2. Agent 筛选出优惠力度大、信用好的商家 Top 10
3. 用户可以同时与**多个**商家 Agent 在四人群聊中谈判
4. 各家 Agent 竞价、提供优惠方案
5. 用户选择最优惠的 → 一键购买

**示例**:
- 小明: "想买跑鞋，预算500-800，喜欢Nike"
- 系统:**立即**找到 15 个相关商家（一对多）
- Agent: 筛选出 5 个有高优惠的商家
- 小明同时与 5 家商家 Agent 在四人群聊中谈判
- Nike Agent: "699减100" → Adidas Agent: "649减150" → ...
- 小明选择最优惠的 → 一键购买

## 通用核心功能

### Agent 层
1. **Agent 创建**: 用户配置自己的 Agent 画像、偏好、边界
2. **Agent 过滤规则**: 地域过滤（同城/同WiFi/N米）、属性过滤（年龄/性别/收入/婚姻）、信用过滤
3. **Agent 通信协议**: 标准化的 Agent 间消息格式，含身份标识（Agent/Human）
4. **Agent 匹配引擎**: 基于意图、位置、画像、过滤规则的智能匹配
5. **Agent 对话**: Agent 之间的多轮对话协商
6. **人机切换**: 用户随时接管 Agent 对话，对方实时感知

### 场景层
7. **VisionShare**: AI 需求解析、本地相册智能检索、隐私脱敏、积分激励
8. **AgentDate**: 兴趣匹配、对话初筛、双向同意机制、真人破冰
9. **AgentJob**: 技能匹配、薪资期望协商、面试安排
10. **AgentAd**: 需求分析、精准推送、优惠协商

### 对话层 (Chat Room)
11. **四人群聊模式**: 甲用户、甲 Agent、乙用户、乙 Agent 在同一个聊天界面
12. **角色标识**: 每条消息清晰显示发送者是谁（用户 or Agent）
13. **人机切换**: 用户随时可切换"亲自聊"或"让 Agent 聊"
14. **Agent 辅助**: Agent 可给用户发送私聊建议，不打扰对方

### 基础层
15. **信用体系**: 用户信用分（0-100）、真人交流后评分、跨场景信用汇总
16. **评分过滤**: 可设置"只与评分>4星"或"信用分>80"的用户交流
17. **隐私控制**: 分级信息披露、用户授权确认、敏感信息 Agent 专用
18. **积分系统**: 跨场景通用积分（查看、响应、匹配成功均有激励）

## 过滤机制详解

### 地域过滤 (Geo Filters) - 按场景配置
```yaml
# VisionShare 场景 - 精确距离
vision_geo:
  distance_range:
    max: 1000              # 1公里内
  same_city: false         # 不强制同城

# AgentDate 场景 - 城市级距离
 dating_geo:
  same_city: true          # 优先同城
  distance_range:
    max: 50000             # 最远50公里
  
# AgentJob 场景 - 城市为主
job_geo:
  same_city: true          # 通常需要同城
  remote_accepted: true    # 但接受远程工作

# AgentAd 场景 - 配送范围
ad_geo:
  same_city: true          # 同城或配送范围内
  delivery_range: 10000    # 10公里配送圈
```

### 属性过滤 (Attribute Filters) - 场景差异化
```yaml
# VisionShare 场景 - 主要关注时效和位置
vision_attributes:
  content_match:
    min_confidence: 0.7      # AI识别置信度 >70%
  photo_quality:
    min_resolution: "1080p"

# AgentDate 场景 - 全面的个人属性
dating_attributes:
  age_range:
    min: 25
    max: 35
  gender: ["female"]
  marital_status: ["single"]           # 只接受未婚
  has_children: false                  # 不接受有子女
  income_range:
    min: 15000
    max: 50000
  has_property: ["has", "either"]      # 有房或无所谓
  education: ["bachelor", "master"]    # 本科及以上
  interests_match:
    min_similarity: 0.5                # 兴趣匹配度 >50%

# AgentJob 场景 - 职业技能相关
job_attributes:
  skills_match:
    required: ["Python", "React"]      # 必需技能
    optional: ["Docker", "K8s"]        # 加分技能
    min_match_ratio: 0.6               # 技能匹配度 >60%
  experience_years:
    min: 3
    max: 8
  salary_overlap:
    min_ratio: 0.5                     # 薪资重叠度 >50%
  company_size: ["startup", "midsize"] # 公司规模偏好
  remote_policy: ["remote", "hybrid"]  # 接受远程/混合办公

# AgentAd 场景 - 消费需求相关
ad_attributes:
  category_match:
    whitelist: ["跑鞋", "运动装备"]    # 只接收这些品类
    blacklist: ["保健品", "金融理财"]  # 不接受这些品类
  price_range:
    min: 500
    max: 800
  brand_preference:
    whitelist: ["Nike", "Adidas", "New Balance"]
    blacklist: ["山寨品牌A", "山寨品牌B"]
  discount_min: 0.2                    # 至少8折才推送
```

### 信用过滤 (Credit Filters)
```yaml
filters:
  credit:
    min_credit_score: 75     # 信用分 >75
    min_avg_rating: 4.0      # 平均评分 >4星
    min_transaction_count: 5 # 完成交易 >5次
    no_complaints: true      # 无投诉记录
    no_reports: true         # 无举报记录
```

## 人机切换与四人群聊机制

### 聊天室模式
BridgeAI 采用"四人群聊"模式，一个聊天室内同时存在四个角色：

```
┌────────────────────────────────────────────────────────────┐
│  💬 BridgeAI 聊天室                                          │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  🤖 甲 Agent (小A)                                         │
│  你好，我是小A，正在帮我的主人了解情况~                      │
│                                                            │
│  🤖 乙 Agent (小B)                                         │
│  你好小A，我是小B，很高兴认识你！                            │
│                                                            │
│  👤 甲用户 (亲自上场)                                        │
│  你好，我是小A的主人，想问一下...                            │
│                                                            │
│  🤖 乙 Agent → 用户                                          │
│  [私聊建议] 主人，对方用户亲自来了，建议真诚回答              │
│                                                            │
│  👤 乙用户 (亲自上场)                                        │
│  你好，我是小B的主人，让我来回答你...                        │
│                                                            │
│  🤖 甲 Agent → 用户                                          │
│  [私聊建议] 主人，对方态度友好，可以深入交流                  │
│                                                            │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  [🤖 让 Agent 代聊]  [👤 亲自聊]  [发送]              │  │
│  └──────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
```

### 消息类型

**群内消息**（所有人可见）：
- 甲 Agent 发送的消息
- 乙 Agent 发送的消息
- 甲用户亲自发送的消息
- 乙用户亲自发送的消息

**私聊消息**（仅自己可见）：
- 自己的 Agent 给自己的建议
- 系统通知（如"对方已切换到 Agent 模式"）

### 切换机制
- **用户切换**: 点击"亲自聊"或"让 Agent 聊"按钮
- **Agent 切换**: Agent 遇到复杂情况主动请求切换
- **实时同步**: 切换状态立即同步给聊天室所有人

### 界面设计
- **头像标识**: Agent 用机器人头像，用户用真人头像
- **名称标识**: "小A (Agent)" / "小A的主人" / "用户昵称"
- **状态标识**: 输入框上方显示当前模式"Agent 代聊中"或"亲自聊天中"
- **私聊区**: 侧边栏或底部显示 Agent 给自己的建议

## 信用与评分系统

### 评分维度
每次真人交流后，双方互评：
- **诚信度** ⭐⭐⭐⭐⭐: 描述是否真实
- **礼貌度** ⭐⭐⭐⭐⭐: 交流是否礼貌
- **响应度** ⭐⭐⭐⭐⭐: 回复是否及时
- **满意度** ⭐⭐⭐⭐⭐: 整体体验
- **场景专属**: VisionShare-照片质量、AgentDate-约会体验、AgentJob-面试专业度

### 信用分计算
```
信用分 = 基础分(60)
       + 完成交易数 × 2
       + 平均评分 × 10
       - 被投诉数 × 5
       - 被举报数 × 10
       + 跨场景信用加成
       
范围: 0-100
等级: 
  - 优秀(90+): 优先推荐，更多功能
  - 良好(75-89): 正常功能
  - 一般(60-74): 限制部分功能
  - 较差(<60): 严格限制，需改善
```

### 评分使用
- **过滤条件**: "只与信用分>80 的用户交流"
- **匹配排序**: 高信用用户优先推荐
- **功能限制**: 低信用用户无法发起某些场景
- **风险提示**: 与低信用用户交流时系统提示

## 场景切换设计

用户在一个 App 中可切换不同场景：
- **首页**: 选择场景图标（VisionShare / AgentDate / AgentJob / AgentAd）
- **Agent 配置**: 统一管理自己的 Agent 画像和过滤规则
- **消息中心**: 所有 Agent 对话和真人引荐集中显示
- **积分钱包**: 跨场景通用积分查看和兑换
- **信用中心**: 查看自己的信用分和评分记录
